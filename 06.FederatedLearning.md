## 1. 연합학습의 필요성

의료, 금융, 모바일 분야의 데이터는 본질적으로 **민감하며 분산되어 있다**. 환자 영상은 각 병원의 서버에, 거래 기록은 각 은행의 데이터베이스에, 사용자 입력 패턴은 각 휴대폰에 저장된다. 이러한 데이터를 하나의 중앙 서버로 모아 학습하면 모델의 성능은 향상되지만, 동시에 **개인정보 유출과 법적 규제 위반의 위험**이 발생한다. 미국의 HIPAA, 유럽의 GDPR, 한국의 개인정보보호법은 모두 민감 데이터의 기관 외부 이동을 엄격히 제한한다.

이러한 배경에서 등장한 것이 **연합학습(Federated Learning)** 이다. 핵심 아이디어는 단순하다. **"데이터를 옮기지 말고, 모델만 옮긴다."** 각 기관은 자신의 데이터로 모델을 학습하고, 그 결과인 가중치만을 중앙 서버에 전송한다. 서버는 여러 기관에서 받은 가중치를 평균하여 하나의 통합 모델을 만들고, 다시 각 기관에 배포한다. 이 과정을 반복하면 데이터는 한 번도 기관 외부로 나가지 않으면서도, 모든 기관의 지식이 통합된 모델을 얻을 수 있다.

도식으로 비교하면 다음과 같다.

```
[기존 중앙집중식]                    [연합학습]

병원A 데이터 ┐                      병원A 모델학습 → 가중치 ┐
병원B 데이터 ├ 중앙서버 → 통합모델   병원B 모델학습 → 가중치 ├ 중앙서버
병원C 데이터 ┘                      병원C 모델학습 → 가중치 ┘ (집계)

데이터 유출 위험                    데이터는 기관에 잔류
법규 위반 가능                      모델 가중치만 공유
```

## 2. 정의와 핵심 최적화 식

연합학습은 **여러 클라이언트가 로컬 데이터로 모델을 학습하고, 그 결과(가중치 또는 그래디언트)만을 중앙 서버에 보내 하나의 글로벌 모델을 만드는 분산 학습 패러다임**으로 정의된다. 클라이언트는 병원, 휴대폰, 은행 등 데이터를 보유한 모든 주체를 의미한다.

수학적으로는 다음과 같은 최적화 문제로 정식화된다.

$$
\min_{w} \ F(w) = \sum_{k=1}^{K} \frac{n_k}{n} F_k(w)
$$

여기서 $K$는 전체 클라이언트의 수이며, $n_k$는 클라이언트 $k$가 보유한 데이터의 수, $n = \sum_k n_k$는 전체 데이터 수이다. $F_k(w)$는 클라이언트 $k$의 로컬 손실함수로, 다음과 같이 정의된다.

$$
F_k(w) = \frac{1}{n_k} \sum_{i \in \mathcal{D}_k} \ell(w; x_i, y_i)
$$

이 식의 핵심은 글로벌 손실이 **각 클라이언트의 로컬 손실을 데이터 크기에 비례하여 가중 평균한 형태**라는 점이다. 이는 모든 데이터를 한 곳에 모아 학습하는 중앙집중식과 수학적으로 동일한 목적을 가지며, 단지 그 계산을 분산된 방식으로 수행할 뿐이다. 데이터를 가상으로 합쳐 학습하는 효과를 내는 것이 연합학습의 본질이라 할 수 있다.

## 3. 전통적 방법론: FedAvg

연합학습의 가장 표준적이고 기본이 되는 알고리즘은 **FedAvg(Federated Averaging)** 이다. 2017년 Google의 McMahan 등이 제안하였으며, 이후 모든 연합학습 알고리즘의 출발점이 된 방법이다. FedAvg의 한 라운드는 다음 네 단계로 구성된다.

**첫째**, 중앙 서버가 현재의 글로벌 모델 $w_t$를 선정된 $K$개의 클라이언트에게 전송한다. **둘째**, 각 클라이언트는 자신의 로컬 데이터로 $E$번의 epoch 동안 SGD를 수행하여 로컬 모델 $w_t^k$를 얻는다. **셋째**, 각 클라이언트는 학습된 가중치 $w_t^k$를 서버로 전송한다. **넷째**, 서버는 받은 가중치들을 데이터 크기에 비례하여 가중 평균한다.

이를 수식으로 표현하면 다음과 같다.

$$
w_{t+1} = \sum_{k=1}^{K} \frac{n_k}{n} \, w_{t+1}^k
$$

이 단순한 가중 평균이 FedAvg의 핵심이다. 이 과정을 수십에서 수백 라운드 반복하면 글로벌 모델이 점차 수렴하게 된다.

FedAvg는 단순함이 가장 큰 장점이지만, 동시에 몇 가지 본질적 한계를 가진다. **첫째**, IID(독립항등분포) 가정에 의존한다. 클라이언트별 데이터 분포가 비슷할 때 잘 작동하지만 그렇지 않은 경우 수렴이 매우 느려진다. **둘째**, 통신 비용이 크다. 매 라운드마다 모델 전체의 가중치를 주고받아야 하므로 모바일 환경에서는 부담이 된다. **셋째**, 클라이언트의 시스템 성능이 다를 때 가장 느린 클라이언트에 전체 속도가 좌우된다.

## 4. Non-IID 문제와 개선 알고리즘

연합학습의 가장 큰 현실적 도전은 **데이터의 비독립항등분포성(Non-IID)** 이다. 일반적인 머신러닝은 모든 학습 데이터가 같은 분포에서 추출되었다고 가정하지만, 연합학습에서는 각 클라이언트의 데이터 분포가 본질적으로 다르다. 예를 들어 A병원은 노인 환자가 다수이고, B병원은 소아 환자가 다수이며, C병원은 특정 희귀질환 환자만 보유한 상황을 생각해볼 수 있다. 이러한 환경에서 단순한 FedAvg는 각 클라이언트의 모델이 서로 다른 방향으로 학습되어, 이를 평균한 결과가 어떤 분포에도 잘 맞지 않는 현상이 발생한다. 이를 **클라이언트 드리프트(client drift)** 라고 한다.

이 문제를 해결하기 위해 다양한 알고리즘이 제안되어 왔다. 대표적인 네 가지를 정리하면 다음과 같다.

| 알고리즘 | 핵심 아이디어 | 출처 |
|---|---|---|
| **FedProx** | 로컬 학습에 proximal term을 추가하여 글로벌 모델에서 너무 멀어지지 않도록 제약 | Li et al., MLSys 2020 |
| **SCAFFOLD** | 클라이언트별 control variate로 드리프트를 직접 보정 | Karimireddy et al., ICML 2020 |
| **FedNova** | 클라이언트마다 다른 로컬 step 수를 정규화하여 평균 | Wang et al., NeurIPS 2020 |
| **MOON** | Contrastive learning으로 로컬·글로벌 representation을 정렬 | Li et al., CVPR 2021 |

이 중 **FedProx**는 가장 단순하면서도 효과적인 접근이다. FedAvg와 다른 점은 로컬 손실함수에 proximal 항을 추가하는 것 한 가지뿐이다.

$$
\min_{w} \ F_k(w) + \frac{\mu}{2} \|w - w_t\|^2
$$

여기서 $\mu$는 proximal 강도를 조절하는 하이퍼파라미터로, 보통 0.001에서 0.1 사이의 값을 사용한다. 이 항의 역할은 로컬 모델 $w$가 글로벌 모델 $w_t$에서 지나치게 벗어나지 못하도록 페널티를 부여하는 것이다. 결과적으로 모든 클라이언트의 로컬 모델이 글로벌 모델 근처에서 학습되어, 평균을 취했을 때 의미 있는 통합 모델이 얻어진다.

**SCAFFOLD**는 더 정교한 방법으로, 각 클라이언트에 control variate $c_k$를, 서버에 $c$를 두어 그래디언트를 직접 보정한다.

$$
w \leftarrow w - \eta \left( \nabla F_k(w) - c_k + c \right)
$$

이를 통해 클라이언트의 그래디언트가 글로벌 방향으로 정렬되며, 이론적으로 IID 환경과 동등한 수렴 속도를 보장한다.

## 5. 최신 방법론

최근 5년간 연합학습 분야는 단순한 분산 학습을 넘어 다양한 방향으로 확장되어 왔다. 본 절에서는 네 가지 주요 흐름을 살펴본다.

### 5-1. Personalized Federated Learning

기존의 FedAvg 계열은 **모든 클라이언트가 하나의 글로벌 모델을 공유**한다는 가정에 기반하였으나, 이는 현실에서 항상 최선이 아니다. 병원마다 환자 구성과 진료 패턴이 다르고, 사용자마다 휴대폰 사용 습관이 다르기 때문이다. 따라서 글로벌 지식을 활용하면서도 **각 클라이언트에 맞춤화된 모델**을 학습하려는 시도가 등장하였다.

대표적인 기법으로는 메타 학습 기반의 **Per-FedAvg**(Fallah et al., NeurIPS 2020), 공유 representation과 개인 head를 분리하는 **FedRep**(Collins et al., ICML 2021), Moreau envelope를 활용한 **pFedMe**(T Dinh et al., NeurIPS 2020) 등이 있다.

pFedMe의 최적화 식은 다음과 같다.

$$
\min_{w} \sum_k \tilde{F}_k(w), \quad \tilde{F}_k(w) = \min_{\theta_k} \left\{ F_k(\theta_k) + \frac{\lambda}{2}\|\theta_k - w\|^2 \right\}
$$

이 식은 **개인 모델 $\theta_k$**와 **공유 모델 $w$**를 동시에 학습하되, 두 모델이 너무 멀어지지 않도록 정규화한다. 결과적으로 각 클라이언트는 글로벌 지식의 혜택을 누리면서도 자신의 데이터에 특화된 모델을 갖게 된다.

### 5-2. Foundation Model과의 결합

2023년 이후 거대언어모델(LLM) 및 foundation model의 등장으로, 연합학습 연구의 새로운 축이 형성되었다. 기존 FedAvg는 모델 전체의 가중치를 매 라운드 주고받아야 했으나, 수십억 파라미터의 LLM에서는 이것이 사실상 불가능하다. 이에 따라 **파라미터 효율적(parameter-efficient) 연합학습**이 주목받고 있다.

대표적인 접근은 **LoRA(Low-Rank Adaptation)와 결합된 연합학습**이다. 글로벌 모델 $W \in \mathbb{R}^{d \times d}$는 고정한 채로, 각 클라이언트는 저차원 행렬 $A_k \in \mathbb{R}^{d \times r}$와 $B_k \in \mathbb{R}^{r \times d}$만 학습한다. 여기서 $r \ll d$이므로 통신해야 할 파라미터가 수백에서 수천 배 감소한다. 서버는 $A_k$와 $B_k$만을 평균하여 통합한다.

**OpenFedLLM**(Ye et al., KDD 2024)은 이러한 LLM 기반 연합학습의 종합 벤치마크를 제공하며, 의료·법률·금융 도메인 특화 LLM을 연합학습으로 fine-tuning한 결과를 보고하였다.

### 5-3. 탈중앙화 연합학습

기존 연합학습은 중앙 서버를 가정하지만, 이는 **단일 실패 지점(single point of failure)** 이 된다는 약점이 있다. 서버가 마비되거나 신뢰할 수 없는 경우 시스템 전체가 작동하지 않는다. 이에 따라 **탈중앙화 연합학습(Decentralized FL)** 이 제안되었다.

대표적으로 **Gossip Learning**과 **BrainTorrent**(Roy et al., MICCAI 2019)는 중앙 서버 없이 클라이언트끼리 직접 가중치를 교환한다. 각 클라이언트는 이웃 클라이언트와 모델을 평균하며, 이 과정이 네트워크 전체에 전파되어 결국 모든 클라이언트가 통합된 모델을 갖게 된다. 이러한 방식은 더 강한 분산성과 견고성을 제공하지만, 수렴 분석과 통신 토폴로지 설계가 복잡해진다는 단점이 있다.

### 5-4. 연합 자기지도학습

의료 영상처럼 **라벨링 비용이 매우 비싼 영역**에서는 라벨이 없는 데이터를 활용한 사전학습이 중요하다. **Federated Self-Supervised Learning**은 각 클라이언트가 자신의 라벨 없는 데이터로 representation을 학습하고, 이를 연합으로 통합하는 방식이다. 대표 기법으로는 **FedSSL**과 **FedX**(Han et al., ECCV 2022)가 있다. 이러한 접근은 향후 의료 영상 기반 연합학습에서 핵심적 역할을 할 것으로 전망된다.

## 6. 프라이버시 강화 기법

연합학습이 데이터를 직접 전송하지 않는다는 사실만으로 **완전한 프라이버시가 보장되지는 않는다**. 모델의 가중치나 그래디언트로부터 원본 데이터를 어느 정도 복원할 수 있다는 연구 결과가 다수 보고되었다. 대표적으로 **Model Inversion Attack**(Fredrikson et al., CCS 2015)은 모델의 출력만으로 학습 데이터의 일부를 재구성할 수 있음을 보였고, **Deep Leakage from Gradients**(Zhu et al., NeurIPS 2019)는 그래디언트로부터 학습 이미지를 거의 완벽히 복원할 수 있음을 입증하였다. 따라서 연합학습은 항상 추가적인 프라이버시 강화 기법과 결합되어 사용된다.

대표적인 네 가지 기법을 정리하면 다음과 같다.

| 기법 | 핵심 원리 | 트레이드오프 |
|---|---|---|
| **Differential Privacy (DP)** | 가중치나 그래디언트에 통계적 노이즈 추가 | 노이즈가 클수록 정확도 저하 |
| **Secure Aggregation** | 서버가 개별 가중치는 보지 못하고 합계만 보도록 암호학적 처리 | 계산 및 통신 비용 증가 |
| **Homomorphic Encryption (HE)** | 암호화된 상태에서 직접 연산 수행 | 매우 느림(10~1000배) |
| **Trusted Execution Environment (TEE)** | 하드웨어 격리 영역에서 학습(Intel SGX 등) | 특정 하드웨어에 의존 |

이 중 **Differential Privacy**는 가장 이론적으로 단단한 보장을 제공한다. 알고리즘 $\mathcal{M}$이 $(\varepsilon, \delta)$-DP를 만족한다는 것은 다음 부등식이 성립함을 의미한다.

$$
\Pr[\mathcal{M}(D) \in S] \leq e^{\varepsilon} \cdot \Pr[\mathcal{M}(D') \in S] + \delta
$$

여기서 $D$와 $D'$는 단 한 개의 샘플만 다른 이웃 데이터셋이고, $\varepsilon$은 프라이버시 예산이며 작을수록 강한 보호를 의미한다. 실무에서는 보통 $\varepsilon$을 0.1에서 10 사이로 설정한다. $\delta$는 극히 작은 실패 확률을 나타낸다.

연합학습에서 가장 널리 쓰이는 구현은 **DP-SGD**(Abadi et al., CCS 2016)이다. 핵심 두 단계는 그래디언트 클리핑과 가우시안 노이즈 주입이다.

```python
g_clipped = g / max(1, ||g||_2 / C)         # 민감도 제한
g_noisy   = g_clipped + N(0, sigma^2 * C^2 * I)  # 노이즈 주입
```

## 7. 실습 코드

본 절에서는 FedAvg를 처음부터 구현하여 IID와 Non-IID 환경에서의 차이를 직접 확인한다. 모델은 단순한 CNN을, 데이터는 MNIST를 사용한다.

### 7-1. 데이터 분할

연합학습 실험의 핵심은 데이터를 어떻게 클라이언트별로 나누느냐이다. IID 분할은 단순 무작위로 균등하게 나누는 반면, Non-IID 분할은 각 클라이언트에 일부 클래스만 할당한다.

```python
import numpy as np
from torch.utils.data import Subset

def split_data_iid(dataset, K):
    """IID 분할: 무작위 균등 분배"""
    idx = np.random.permutation(len(dataset))
    return [Subset(dataset, idx[i::K]) for i in range(K)]

def split_data_non_iid(dataset, K, classes_per_client=2):
    """Non-IID 분할: 각 클라이언트가 일부 클래스만 보유"""
    labels = np.array([dataset[i][1] for i in range(len(dataset))])
    client_data = [[] for _ in range(K)]
    
    for c in range(10):
        idx_c = np.where(labels == c)[0]
        np.random.shuffle(idx_c)
        chosen = np.random.choice(K, classes_per_client, replace=False)
        splits = np.array_split(idx_c, classes_per_client)
        for client_id, split in zip(chosen, splits):
            client_data[client_id].extend(split.tolist())
    
    return [Subset(dataset, idx) for idx in client_data]
```

### 7-2. 모델 정의

```python
import torch.nn as nn
import torch.nn.functional as F

class SimpleCNN(nn.Module):
    def __init__(self, n_classes=10):
        super().__init__()
        self.conv1 = nn.Conv2d(1, 16, 3, padding=1)
        self.conv2 = nn.Conv2d(16, 32, 3, padding=1)
        self.fc1   = nn.Linear(32 * 7 * 7, 64)
        self.fc2   = nn.Linear(64, n_classes)
    
    def forward(self, x):
        x = F.max_pool2d(F.relu(self.conv1(x)), 2)
        x = F.max_pool2d(F.relu(self.conv2(x)), 2)
        x = x.flatten(1)
        x = F.relu(self.fc1(x))
        return self.fc2(x)
```

### 7-3. 로컬 학습과 가중 평균

여기가 FedAvg의 핵심이다. 각 클라이언트가 자신의 데이터로 학습한 후, 서버가 데이터 크기로 가중 평균을 취한다.

```python
import torch
import copy
from torch.utils.data import DataLoader

def local_train(model, dataloader, epochs=2, lr=0.01):
    """한 클라이언트의 로컬 학습"""
    optimizer = torch.optim.SGD(model.parameters(), lr=lr)
    model.train()
    for _ in range(epochs):
        for x, y in dataloader:
            optimizer.zero_grad()
            loss = F.cross_entropy(model(x), y)
            loss.backward()
            optimizer.step()
    return model.state_dict()


def federated_averaging(local_states, weights):
    """
    FedAvg의 핵심: 가중 평균
    - local_states: K개 클라이언트의 state_dict 리스트
    - weights:      각 클라이언트의 데이터 수
    """
    total = sum(weights)
    avg_state = copy.deepcopy(local_states[0])
    
    for key in avg_state.keys():
        avg_state[key] = sum(
            local_states[k][key] * (weights[k] / total)
            for k in range(len(local_states))
        )
    return avg_state
```

### 7-4. 한 라운드 실행

```python
def federated_round(global_model, client_datasets, local_epochs=2):
    local_states = []
    weights = []
    
    for dataset in client_datasets:
        local_model = copy.deepcopy(global_model)
        loader = DataLoader(dataset, batch_size=32, shuffle=True)
        local_state = local_train(local_model, loader, epochs=local_epochs)
        local_states.append(local_state)
        weights.append(len(dataset))
    
    new_state = federated_averaging(local_states, weights)
    global_model.load_state_dict(new_state)
    return global_model
```

### 7-5. 전체 학습 루프

```python
from torchvision import datasets, transforms

def run_federated_learning(num_clients=10, num_rounds=20, non_iid=False):
    transform = transforms.Compose([transforms.ToTensor()])
    train_data = datasets.MNIST('./data', train=True, download=True, transform=transform)
    test_data  = datasets.MNIST('./data', train=False, transform=transform)
    
    if non_iid:
        client_datasets = split_data_non_iid(train_data, num_clients)
        print("Non-IID 분할로 학습 시작")
    else:
        client_datasets = split_data_iid(train_data, num_clients)
        print("IID 분할로 학습 시작")
    
    global_model = SimpleCNN()
    test_loader = DataLoader(test_data, batch_size=256)
    
    for t in range(num_rounds):
        global_model = federated_round(global_model, client_datasets, local_epochs=2)
        
        global_model.eval()
        correct = 0
        with torch.no_grad():
            for x, y in test_loader:
                correct += (global_model(x).argmax(1) == y).sum().item()
        acc = correct / len(test_data)
        print(f"Round {t+1:2d} | Test Accuracy: {acc:.4f}")
    
    return global_model
```

이 코드를 IID와 Non-IID로 각각 실행해 보면, **IID 환경에서는 빠르게 95% 이상의 정확도에 수렴하지만, Non-IID 환경에서는 학습이 불안정하고 최종 정확도도 낮음**을 확인할 수 있다. 이것이 바로 FedProx 같은 개선 알고리즘이 필요한 이유이다.

### 7-6. FedProx로의 확장

FedAvg를 FedProx로 바꾸는 데에는 로컬 학습 함수에 proximal 항을 추가하는 단 몇 줄의 코드만 필요하다.

```python
def local_train_fedprox(model, global_state, dataloader, epochs=2, lr=0.01, mu=0.01):
    """FedAvg + proximal term"""
    optimizer = torch.optim.SGD(model.parameters(), lr=lr)
    model.train()
    
    for _ in range(epochs):
        for x, y in dataloader:
            optimizer.zero_grad()
            loss = F.cross_entropy(model(x), y)
            
            # 핵심: proximal term 추가
            prox_term = 0.0
            for name, param in model.named_parameters():
                prox_term += ((param - global_state[name]) ** 2).sum()
            loss = loss + (mu / 2) * prox_term
            
            loss.backward()
            optimizer.step()
    return model.state_dict()
```

실제 산업 환경에서는 직접 구현보다 검증된 프레임워크를 사용한다. 가장 널리 쓰이는 것은 **Flower**(flower.dev)이다. 위의 FedAvg 구현을 Flower로 다시 작성하면 다음과 같다.

import flwr as fl

class MnistClient(fl.client.NumPyClient):
    def __init__(self, model, train_data, test_data):
        self.model = model
        self.train_data = train_data
        self.test_data = test_data
    
    def get_parameters(self, config):
        return [val.cpu().numpy() for _, val in self.model.state_dict().items()]
    
    def fit(self, parameters, config):
        self.set_parameters(parameters)
        train(self.model, self.train_data, epochs=2)
        return self.get_parameters({}), len(self.train_data), {}
    
    def evaluate(self, parameters, config):
        self.set_parameters(parameters)
        loss, accuracy = test(self.model, self.test_data)

## 9. 인공지능 신뢰성과의 연결

연합학습은 본 강의의 다른 신뢰성 모듈들과 본질적으로 얽혀 있다. **프라이버시** 측면에서는 연합학습 자체가 가장 직접적인 해법이며, DP와 Secure Aggregation의 결합으로 강화된다. **공정성** 측면에서는 클라이언트 간 데이터 분포가 다르기 때문에 글로벌 모델이 일부 그룹에 편향될 수 있어, group fairness 연구가 활발히 진행되고 있다. **강건성** 측면에서는 악의적 클라이언트가 잘못된 가중치를 전송하는 **Byzantine attack**과 모델 포이즈닝에 대한 방어가 핵심 연구 주제이다. **불확실성** 측면에서는 클라이언트 간 의견 불일치가 자연스러운 epistemic uncertainty의 측정 도구로 활용될 수 있다.

이러한 의미에서 연합학습은 **인공지능 신뢰성의 네 기둥을 모두 한 번에 다루는 종합 케이스**라 할 수 있으며, 본 강의의 마지막 모듈로 적합하다.
