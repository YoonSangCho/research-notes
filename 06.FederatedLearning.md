## 1. 배경: 인공지능 신뢰성과 프라이버시

본 강의는 인공지능 모델이 갖추어야 할 **신뢰성(trustworthiness)** 을 여섯 가지 축으로 정리한다. 곧 (1) 정확성, (2) 공정성(그룹별 편차 최소화), (3) 재현성(외부평가 강건성), (4) 안정성(공격 및 방어), (5) 신뢰성(불확실성), 그리고 (6) **프라이버시**이다. 연합학습은 이 중 마지막 축인 프라이버시를 직접적으로 다루는 방법론이다.

의료, 금융, 모바일 분야의 데이터는 본질적으로 **민감하며 분산되어 있다**. 환자 영상은 각 병원의 서버에, 거래 기록은 각 은행의 데이터베이스에, 사용자 입력 패턴은 각 휴대폰에 저장된다. 병원의 의무기록(EMR)에는 환자 ID, 병원 ID, 인종, 성별, 주소와 같은 식별 정보가 함께 담겨 있고, 스마트폰의 앱 메시지에는 개인의 사적 입력이 누적된다. 사람·기관·기업에게 **자신의 데이터는 곧 자산**이며, 따라서 외부 유출을 매우 꺼린다. 그 결과 가장 가치 있는 데이터를 정작 모델 학습에 활용하지 못하는 아쉬움이 생긴다.

이러한 데이터를 하나의 중앙 서버로 모아 학습하면 모델의 성능은 향상되지만, 동시에 **개인정보 유출과 법적 규제 위반의 위험**이 발생한다. 미국의 HIPAA, 유럽의 GDPR, 한국의 개인정보보호법은 모두 민감 데이터의 기관 외부 이동을 엄격히 제한한다.

여기서 핵심 질문이 도출된다. **"프라이버시를 보존하면서 동시에 데이터를 활용할 수는 없는가?"** 이 질문에 대한 대표적인 답이 바로 **연합학습(Federated Learning)** 이다.

## 2. 연합학습의 정의와 핵심 아이디어

연합학습은 다음과 같이 정의된다.

> "Federated Learning (also known as collaborative learning) is a machine learning paradigm where multiple clients (e.g., devices or organizations) collaboratively train a model **without sharing their data**."

즉, **여러 클라이언트(디바이스 또는 기관)가 데이터를 공유하지 않고 협력적으로 하나의 모델을 학습하는 패러다임**이다. 핵심 아이디어는 단순하다. **"데이터를 옮기지 말고, 모델만 옮긴다."** 각 기관은 자신의 데이터로 모델을 학습하고, 그 결과인 가중치만을 중앙 서버에 전송한다. 서버는 여러 기관에서 받은 가중치를 평균하여 하나의 통합 모델을 만들고, 다시 각 기관에 배포한다. 이 과정을 반복하면 데이터는 한 번도 기관 외부로 나가지 않으면서도, 모든 기관의 지식이 통합된 모델을 얻을 수 있다.

구조는 두 가지 구성 요소로 나뉜다.

- **Local AI Model (로컬 모델)** — 각 디바이스/클라이언트가 자신의 데이터로 *로컬에서* 학습한다. 데이터는 절대 기관 외부로 나가지 않는다.
- **Global AI Model (글로벌 모델)** — 중앙 서버는 데이터가 아닌 *모델만 집계(aggregation)* 하여 하나의 통합 모델을 만든다.

```
[기존 중앙집중식]                    [연합학습]

병원A 데이터 ┐                      병원A 모델학습 → w_A ┐
병원B 데이터 ├ 중앙서버 → 통합모델   병원B 모델학습 → w_B ├ 중앙서버 (집계)
병원C 데이터 ┘                      병원C 모델학습 → w_C ┘  → 글로벌 모델

데이터 유출 위험                    데이터는 기관에 잔류
법규 위반 가능                      모델 가중치만 공유
```

이 구조가 주는 이점은 명확하다. 개인정보 보호는 물론, 데이터 저장 비용·이동 비용·유지보수 비용이 모두 감소하여 **경제적**이다. 또한 다양한 개인/기업/그룹/기관이 참여할수록 더 많은 데이터의 효과를 누려 **더 좋은 모델**을 도출할 수 있다. 데이터 보안 문제로 협력을 꺼리던 기관들을 설득하기에 좋은, "데이터는 지키면서 빅데이터 효과는 누리는" 방법인 셈이다.

주요 활용 영역은 **의료(EMR)·금융·제조·스마트폰** 등 프라이버시에 민감한 분야이며, 특히 여러 기관을 "다기관 통합"하기 위한 수단으로 쓰인다.

- **금융** — 여러 은행이 데이터를 공유하지 않고 모델 업데이트만 교환하여 사기 탐지, 신용 평가 모델을 공동 구축.
- **제조** — 경쟁·프라이버시 문제로 데이터 공유가 어려운 제조업체들이 협력 학습으로 예측 유지보수(장비 고장 사전 예측)와 품질 관리 모델을 구축.
- **의료** — 병원 간 환자 데이터 공유가 제한된 상황에서, MRI·CT 등 의료 영상 기반 진단 모델과 개인 맞춤형 치료 모델을 공동 학습.

## 3. 핵심 최적화 식과 학습 원리

### 3-1. 최적화 목적식

연합학습의 기본 목표는 **글로벌 모델 $w_{\text{global}}$** 을 학습하는 것이며, 다음 최적화 문제로 정식화된다.

$$
\min_{w_{\text{global}}} \ F(w_{\text{global}}) = \sum_{k=1}^{K} \frac{n_k}{N} \, L_k\!\left(w_{\text{global}}\right)
$$

각 기호의 의미는 다음과 같다.

- $w_{\text{global}}$ : 글로벌 모델 (간단히 $w$로 표기하기도 함)
- $k$ : 클라이언트 단위, $K$ : 전체 클라이언트 수
- $w_k$ : 클라이언트 $k$의 로컬 모델
- $n_k$ : 클라이언트 $k$의 데이터 수, $N = \sum_k n_k$ : 전체 데이터 수
- $L_k(w_{\text{global}})$ : 클라이언트 $k$의 데이터에 대한 로컬 손실 함수

$L_k$는 더 구체적으로 다음과 같이 쓸 수 있다.

$$
L_k(w) = \frac{1}{n_k} \sum_{i \in \mathcal{D}_k} \ell(w; x_i, y_i)
$$

이 식의 핵심은 글로벌 손실이 **각 클라이언트의 로컬 손실을 데이터 크기 비율 $n_k/N$로 가중 평균한 형태**라는 점이다. 이는 모든 데이터를 한 곳에 모아 학습하는 중앙집중식과 수학적으로 동일한 목적을 가지며, 단지 그 계산을 분산된 방식으로 수행할 뿐이다. **데이터를 가상으로 합쳐 학습하는 효과**를 내는 것이 연합학습의 본질이라 할 수 있다.

### 3-2. 학습 원리: 한 라운드의 5단계

연합학습은 다음 다섯 단계를 한 라운드로 하여 여러 번 반복한다.

1. **초기 모델 배포** — 중앙 서버가 글로벌 모델 $w_{\text{global}}^{t}$를 클라이언트들에게 전달한다.
2. **클라이언트별 로컬 학습** — 각 클라이언트는 자신의 데이터로 로컬 모델을 학습(로컬 SGD)한다.
   $$
   w_k^{t+1} = w_k^{t} - \eta \, \nabla L_k\!\left(w_k^{t}\right)
   $$
3. **모델 업데이트 수집** — 서버는 클라이언트가 갱신한 로컬 모델 $w_k$를 수집한다.
4. **글로벌 모델 갱신** — 평균 또는 다른 방식으로 집계(aggregation)하여 글로벌 모델을 갱신한다.
   $$
   w_{\text{global}}^{t+1} = \sum_{k=1}^{K} p_k \, w_k^{t+1}, \qquad p_k = \frac{n_k}{N}
   $$
5. 위 과정을 **여러 라운드 반복**한다.

여기서 **(2) 로컬 업데이트(client update)** 와 **(4) 글로벌 업데이트(server aggregation)** 의 구분이 모든 연합학습 알고리즘을 이해하는 기본 골격이 된다. 아래에서 살펴볼 네 가지 전통적 방법론은 모두 이 두 단계의 변형으로 정리할 수 있다.

## 4. 전통적 방법론: FedAvg · FedProx · FedNova · FedOpt

본 절에서는 연합학습의 대표적이고 표준적인 네 가지 방법론을 다룬다. 핵심은 **"로컬 업데이트를 어떻게 할 것인가"** 와 **"서버에서 어떻게 집계할 것인가"** 의 조합이다.

| 방법론 | 변형 위치 | 핵심 아이디어 | 출처 |
|---|---|---|---|
| **FedAvg** | 집계 | 로컬 SGD 후 단순(가중) 평균 | McMahan et al., AISTATS 2017 |
| **FedProx** | 로컬 | proximal 항으로 글로벌 모델 근방 학습 | Li et al., MLSys 2020 |
| **FedNova** | 집계 | 로컬 step 수 차이를 정규화하여 평균 | Wang et al., NeurIPS 2020 |
| **FedOpt** | 집계 | 서버에서 적응적 옵티마이저(Adam 등) 적용 | Reddi et al., ICLR 2021 |

### 4-1. FedAvg (Federated Averaging)

연합학습의 가장 표준적이고 기본이 되는 알고리즘이다. 2017년 Google의 McMahan 등이 제안하였으며, 이후 모든 연합학습 알고리즘의 출발점이 된 방법이다. **각 클라이언트에서 로컬 SGD를 수행하고 서버에서 단순(가중) 평균하여 모델을 업데이트**한다.

FedAvg의 한 라운드는 다음 네 단계로 구성된다. **첫째**, 중앙 서버가 현재의 글로벌 모델 $w_t$를 선정된 $K$개의 클라이언트에게 전송한다. **둘째**, 각 클라이언트는 자신의 로컬 데이터로 $E$번의 epoch 동안 SGD를 수행하여 로컬 모델 $w_k^{t+1}$를 얻는다. **셋째**, 각 클라이언트는 학습된 가중치를 서버로 전송한다. **넷째**, 서버는 받은 가중치들을 데이터 크기에 비례하여 가중 평균한다.

$$
\textbf{Local: } \quad w_k^{t+1} = w_k^{t} - \eta \, \nabla L_k\!\left(w_k^{t}\right)
$$

$$
\textbf{Global: } \quad w_{\text{global}}^{t+1} = \sum_{k=1}^{K} p_k \, w_k^{t+1}, \quad p_k = \frac{n_k}{N}
$$

이 단순한 가중 평균이 FedAvg의 핵심이다. 이 과정을 수십에서 수백 라운드 반복하면 글로벌 모델이 점차 수렴하게 된다.

FedAvg는 단순함이 가장 큰 장점이지만, 동시에 몇 가지 본질적 한계를 가진다. **첫째**, IID(독립항등분포) 가정에 의존한다. 클라이언트별 데이터 분포가 비슷할 때 잘 작동하지만 그렇지 않은 경우 수렴이 매우 느려진다. **둘째**, 통신 비용이 크다. 매 라운드마다 모델 전체의 가중치를 주고받아야 하므로 모바일 환경에서는 부담이 된다. **셋째**, 클라이언트의 시스템 성능이 다를 때 가장 느린 클라이언트에 전체 속도가 좌우된다. 이 한계들이 다음 세 방법론이 등장한 동기이다.

### 4-2. FedProx (Federated Proximal Optimization)

FedAvg의 확장 버전으로, **Non-IID 환경의 문제를 완화하기 위해** 설계되었다. 핵심 아이디어는 **"글로벌 모델 근처에서만 학습하자"** 이다. 로컬 최적화 과정에서 클라이언트가 지나치게 제각기 다른 방향으로 학습되지 않도록, 글로벌 모델과의 차이를 최소화하는 **규제항(regularization)** 을 추가한다. 이를 **Proximal Term(근방 항)** 이라 한다.

$$
\min_{w_k}\ L_k\!\left(w_k\right) + \frac{\mu}{2}\,\bigl\lVert w_k - w_{\text{global}} \bigr\rVert^2
$$

- $L_k(w_k)$ : 클라이언트 $k$의 로컬 손실 (local objective)
- $w_k$ : 클라이언트의 로컬 모델 파라미터
- $w_{\text{global}}$ : 서버로부터 받은 글로벌 모델 파라미터
- $\mu$ : 정규화(유사성) 강도를 조절하는 하이퍼파라미터 (보통 0.001~0.1)

추가되는 것은 proximal 항 단 하나뿐이며, 글로벌 업데이트는 FedAvg와 동일한 가중 평균을 사용한다. 결과적으로 모든 로컬 모델이 글로벌 모델 근방에 머물러, 평균을 취했을 때 안정적인 통합 모델을 얻는다.

### 4-3. FedNova (Normalized Averaging)

FedNova는 **클라이언트마다 로컬 학습의 step 수가 다를 때 생기는 불일치(objective inconsistency)** 를 해결한다. 어떤 클라이언트는 데이터가 많아 한 라운드에 수백 step을 돌고, 어떤 클라이언트는 수십 step만 도는 상황을 생각해 보자. FedAvg로 단순 평균하면 step을 많이 돈 클라이언트가 과도하게 반영되어 학습 목표가 왜곡된다.

FedNova는 각 클라이언트의 누적 로컬 업데이트를 **자신의 로컬 step 수 $\tau_k$로 정규화한 뒤** 평균하고, 이를 유효 step 수 $\tau_{\text{eff}}$로 다시 스케일한다.

$$
w_{\text{global}}^{t+1} = w_{\text{global}}^{t} - \tau_{\text{eff}} \sum_{k=1}^{K} p_k \, \frac{\Delta_k}{\tau_k}, \qquad
\tau_{\text{eff}} = \sum_{k=1}^{K} p_k \, \tau_k
$$

여기서 $\Delta_k = w_{\text{global}}^{t} - w_k^{t+1}$는 클라이언트 $k$의 총 로컬 변화량, $\tau_k$는 로컬 step 수이다. step 수의 편차로 인한 편향을 제거하여, 이질적(heterogeneous) 환경에서도 일관된 목표로 수렴하도록 만든다.

### 4-4. FedOpt (Adaptive Federated Optimization)

FedOpt는 **글로벌 업데이트(서버 집계) 단계에 적응적 옵티마이저를 도입**한 일반화된 프레임워크이다. 핵심 통찰은 서버가 받은 평균 업데이트를 **유사 그래디언트(pseudo-gradient)** 로 간주하는 것이다.

$$
\Delta_t = \sum_{k=1}^{K} p_k \left(w_k^{t+1} - w_{\text{global}}^{t}\right)
$$

서버는 이 $\Delta_t$를 마치 그래디언트처럼 사용하여, 단순 평균 대신 Adam·Yogi·Adagrad 같은 적응적 옵티마이저로 글로벌 모델을 갱신한다. 예컨대 **FedAdam**은 다음과 같다.

$$
m_t = \beta_1 m_{t-1} + (1-\beta_1)\,\Delta_t, \qquad
v_t = \beta_2 v_{t-1} + (1-\beta_2)\,\Delta_t^{2}
$$

$$
w_{\text{global}}^{t+1} = w_{\text{global}}^{t} + \eta \, \frac{m_t}{\sqrt{v_t} + \tau}
$$

FedAvg는 서버 옵티마이저가 학습률 1의 SGD인 특수한 경우에 해당한다. 적응적 모멘텀·분산 보정을 통해 수렴을 가속하고 안정화하며, 통신 라운드 수를 줄이는 효과가 있다.

## 5. Non-IID 문제와 추가 개선 알고리즘

연합학습의 가장 큰 현실적 도전은 **데이터의 비독립항등분포성(Non-IID)** 이다. 일반적인 머신러닝은 모든 학습 데이터가 같은 분포에서 추출되었다고 가정하지만, 연합학습에서는 각 클라이언트의 데이터 분포가 본질적으로 다르다. 예를 들어 A병원은 노인 환자가 다수이고, B병원은 소아 환자가 다수이며, C병원은 특정 희귀질환 환자만 보유한 상황을 생각해볼 수 있다. 이러한 환경에서 단순한 FedAvg는 각 클라이언트의 모델이 서로 다른 방향으로 학습되어, 이를 평균한 결과가 어떤 분포에도 잘 맞지 않는 현상이 발생한다. 이를 **클라이언트 드리프트(client drift)** 라고 한다.

앞서 다룬 FedProx·FedNova·FedOpt 외에도, 다양한 알고리즘이 이 문제를 해결하기 위해 제안되었다. 추가적으로 알아둘 만한 두 가지를 정리하면 다음과 같다.

| 알고리즘 | 변형 위치 | 핵심 아이디어 | 출처 |
|---|---|---|---|
| **SCAFFOLD** | 로컬 | 클라이언트별 control variate로 드리프트를 직접 보정 | Karimireddy et al., ICML 2020 |
| **FedDyn** | 로컬 | 동적 정규화로 로컬 최적점을 글로벌 정상점과 점근적으로 일치 | Acar et al., ICLR 2021 |
| **FedAvgM** | 집계 | 서버 측 모멘텀(server momentum)으로 집계 안정화 | Hsu et al., 2019 |
| **MOON** | 로컬 | Contrastive learning으로 로컬·글로벌 representation을 정렬 | Li et al., CVPR 2021 |

**SCAFFOLD**는 정교한 방법으로, 각 클라이언트에 control variate $c_k$를, 서버에 $c$를 두어 그래디언트를 직접 보정한다.

$$
w \leftarrow w - \eta \left( \nabla L_k(w) - c_k + c \right)
$$

여기서 $c$는 글로벌 평균 방향, $c_k$는 클라이언트 $k$의 로컬 방향을 추정한 보정항이다. $-c_k + c$ 항이 로컬 그래디언트를 글로벌 방향으로 끌어당겨 드리프트를 상쇄하며, 이론적으로 IID 환경과 동등한 수렴 속도를 보장한다. **FedDyn**은 다른 접근으로, 각 라운드마다 로컬 손실에 동적으로 갱신되는 선형·이차 정규화 항을 더해, 로컬 최적해가 점근적으로 글로벌 손실의 정상점과 일치하도록 만든다. SCAFFOLD가 추가 control variate를 통신해야 하는 반면 FedDyn은 추가 통신 없이 드리프트를 줄인다는 차이가 있다.

## 6. 연합학습 프레임워크 I — 통신 구조 관점

연합학습은 학습 과정에서 데이터를 공유하지 않되, **모델 업데이트를 어떤 통신 구조로 주고받는가**에 따라 두 가지로 구분된다.

| 구분 | 구조 | 특징 | 대표 방법 |
|---|---|---|---|
| **Centralized FL** | 중앙 서버 존재 | 모든 클라이언트가 서버에 모델 업데이트를 전송하고, 서버가 집계·재배포 | FedAvg, FedProx |
| **Decentralized FL** | 중앙 서버 없음 | 클라이언트들끼리 직접(peer-to-peer) 모델을 교환. 이웃 클라이언트와 업데이트를 주고받으며 학습이 네트워크 전체로 전파 | Gossip Learning, BrainTorrent, blockchain-based FL |

중앙집중식은 구현이 단순하고 집계가 명확하지만, 서버가 **단일 실패 지점(single point of failure)** 이 된다는 약점이 있다. 서버가 마비되거나 신뢰할 수 없는 경우 시스템 전체가 작동하지 않는다.

탈중앙화 방식은 서버 의존성을 없애 견고성과 분산성을 높이지만, 수렴 분석과 통신 토폴로지 설계가 복잡해진다. 대표적으로 **Gossip Learning**과 **BrainTorrent**(Roy et al., MICCAI 2019)는 중앙 서버 없이 클라이언트끼리 직접 가중치를 교환한다. 각 클라이언트는 이웃 클라이언트와 모델을 평균하며, 이 과정이 네트워크 전체에 전파되어 결국 모든 클라이언트가 통합된 모델을 갖게 된다.

## 7. 연합학습 프레임워크 II — 데이터 분포 관점

또 다른 분류 축은 클라이언트들이 **데이터를 어떤 방식으로 분할하여 보유하는가**이다. 같은 표를 행(sample)과 열(feature)로 나눈다고 생각하면 직관적이다.

| 유형 | 데이터 특징 | 예시 | 대표 방법 |
|---|---|---|---|
| **Horizontal FL** (수평) | 같은 feature space, 서로 다른 sample 집합 | 여러 병원이 같은 검사 항목(혈압·체온)을 보유하나 환자군이 다름 | FedAvg, FedProx |
| **Vertical FL** (수직) | 같은 sample 집합, 서로 다른 feature 보유 | 병원 A는 혈압·심박수, 보험사 B는 소득·보험료 — 환자는 동일 | SplitNN, SecureBoost |
| **Federated Transfer Learning** (FTL) | sample도 feature도 다르되 일부 sample이 중복 | 한국 병원 A와 미국 병원 B의 환자는 거의 겹치지 않으나 일부 중복 | FTL (Yang et al., 2019), 지식 증류(KD) 기반 FL |

- **Horizontal FL** — 가장 쉬운 경우이다. 모든 클라이언트가 동일한 모델 구조로 학습할 수 있어 FedAvg/FedProx가 그대로 적용된다.
- **Vertical FL** — feature가 기관마다 나뉘어 있어, feature-level joint learning을 위한 **암호화 기반 계산**과 **secure entity alignment**(어떤 sample이 동일 개체인지 안전하게 매칭)가 필요하다. 각자 로컬 feature를 forward한 뒤 중앙에서 결합하여 backprop하는 분할 학습 구조를 쓴다.
- **Federated Transfer Learning** — 중복되는 작은 subset을 기반으로 **knowledge transfer**(fine-tuning, domain adaptation)를 수행하여, 데이터가 거의 겹치지 않는 기관 간에도 협력 학습을 가능하게 한다.

## 8. 최신 방법론

최근 5년간 연합학습 분야는 단순한 분산 학습을 넘어 다양한 방향으로 확장되어 왔다. 본 절에서는 세 가지 주요 흐름을 살펴본다.

### 8-1. Personalized Federated Learning

기존의 FedAvg 계열은 **모든 클라이언트가 하나의 글로벌 모델을 공유**한다는 가정에 기반하였으나, 이는 현실에서 항상 최선이 아니다. 병원마다 환자 구성과 진료 패턴이 다르고, 사용자마다 휴대폰 사용 습관이 다르기 때문이다. 따라서 글로벌 지식을 활용하면서도 **각 클라이언트에 맞춤화된 모델**을 학습하려는 시도가 등장하였다.

대표적인 기법으로는 메타 학습 기반의 **Per-FedAvg**(Fallah et al., NeurIPS 2020), 공유 representation과 개인 head를 분리하는 **FedRep**(Collins et al., ICML 2021), Moreau envelope를 활용한 **pFedMe**(T Dinh et al., NeurIPS 2020) 등이 있다.

pFedMe의 최적화 식은 다음과 같다.

$$
\min_{w} \sum_k \tilde{F}_k(w), \quad \tilde{F}_k(w) = \min_{\theta_k} \left\{ L_k(\theta_k) + \frac{\lambda}{2}\|\theta_k - w\|^2 \right\}
$$

이 식은 **개인 모델 $\theta_k$**와 **공유 모델 $w$**를 동시에 학습하되, 두 모델이 너무 멀어지지 않도록 정규화한다. 결과적으로 각 클라이언트는 글로벌 지식의 혜택을 누리면서도 자신의 데이터에 특화된 모델을 갖게 된다.

### 8-2. Foundation Model과의 결합

2023년 이후 거대언어모델(LLM) 및 foundation model의 등장으로, 연합학습 연구의 새로운 축이 형성되었다. 기존 FedAvg는 모델 전체의 가중치를 매 라운드 주고받아야 했으나, 수십억 파라미터의 LLM에서는 이것이 사실상 불가능하다. 이에 따라 **파라미터 효율적(parameter-efficient) 연합학습**이 주목받고 있다.

대표적인 접근은 **LoRA(Low-Rank Adaptation)와 결합된 연합학습**이다. 글로벌 모델 $W \in \mathbb{R}^{d \times d}$는 고정한 채로, 각 클라이언트는 저차원 행렬 $A_k \in \mathbb{R}^{d \times r}$와 $B_k \in \mathbb{R}^{r \times d}$만 학습한다. 여기서 $r \ll d$이므로 통신해야 할 파라미터가 수백에서 수천 배 감소한다. 서버는 $A_k$와 $B_k$만을 평균하여 통합한다.

**OpenFedLLM**(Ye et al., KDD 2024)은 이러한 LLM 기반 연합학습의 종합 벤치마크를 제공하며, 의료·법률·금융 도메인 특화 LLM을 연합학습으로 fine-tuning한 결과를 보고하였다.

### 8-3. 연합 자기지도학습

의료 영상처럼 **라벨링 비용이 매우 비싼 영역**에서는 라벨이 없는 데이터를 활용한 사전학습이 중요하다. **Federated Self-Supervised Learning**은 각 클라이언트가 자신의 라벨 없는 데이터로 representation을 학습하고, 이를 연합으로 통합하는 방식이다. 대표 기법으로는 **FedSSL**과 **FedX**(Han et al., ECCV 2022)가 있다. 이러한 접근은 향후 의료 영상 기반 연합학습에서 핵심적 역할을 할 것으로 전망된다.

## 9. 프라이버시 강화 기법

연합학습이 데이터를 직접 전송하지 않는다는 사실만으로 **완전한 프라이버시가 보장되지는 않는다**. 모델의 가중치나 그래디언트로부터 원본 데이터를 어느 정도 복원할 수 있다는 연구 결과가 다수 보고되었다. 대표적으로 **Model Inversion Attack**(Fredrikson et al., CCS 2015)은 모델의 출력만으로 학습 데이터의 일부를 재구성할 수 있음을 보였고, **Deep Leakage from Gradients (DLG)** (Zhu et al., NeurIPS 2019)는 그래디언트로부터 학습 이미지를 거의 완벽히 복원할 수 있음을 입증하였다. 후속 연구인 **iDLG**(Zhao et al., 2020)는 교차 엔트로피 손실에서 마지막 층 그래디언트의 **부호만으로 정답 라벨을 해석적으로 추출**할 수 있음을 보여, 입력 복원만 최적화하면 되도록 만들어 DLG보다 안정적이고 빠르게 수렴한다. 이후 **Inverting Gradients**(Geiping et al., NeurIPS 2020)는 더 깊은 네트워크와 배치 단위에서도 고해상도 영상을 복원하여 위협의 현실성을 한층 높였다. 따라서 연합학습은 항상 추가적인 프라이버시 강화 기법과 결합되어 사용된다.

대표적인 다섯 가지 기법을 정리하면 다음과 같다.

| 기법 | 핵심 원리 | 트레이드오프 | 대표 적용 |
|---|---|---|---|
| **Differential Privacy (DP)** | 가중치나 그래디언트에 통계적 노이즈 추가 | 노이즈가 클수록 정확도 저하 | Google, Apple |
| **Secure Aggregation** | 서버가 개별 값은 보지 못하고 합계만 보도록 암호학적 처리 | 계산 및 통신 비용 증가 | Google, Apple |
| **Homomorphic Encryption (HE)** | 암호화된 상태에서 직접 연산 수행 | 매우 느림(10~1000배) | Intel + UPenn 의료 |
| **Trusted Execution Environment (TEE)** | 하드웨어 격리 영역에서 학습(Intel SGX 등) | 특정 하드웨어에 의존 | — |
| **Gradient Compression** | Top-k sparsification 등으로 통신량 절감 | 정확도 약간 손실 가능 | Amazon (edge) |

### 9-1. Differential Privacy 상세

**Differential Privacy**는 가장 이론적으로 단단한 보장을 제공한다. 알고리즘 $\mathcal{M}$이 $(\varepsilon, \delta)$-DP를 만족한다는 것은 다음 부등식이 성립함을 의미한다.

$$
\Pr[\mathcal{M}(D) \in S] \leq e^{\varepsilon} \cdot \Pr[\mathcal{M}(D') \in S] + \delta
$$

여기서 $D$와 $D'$는 단 한 개의 샘플만 다른 이웃 데이터셋이고, $\varepsilon$은 프라이버시 예산이며 작을수록 강한 보호를 의미한다. 실무에서는 보통 $\varepsilon$을 0.1에서 10 사이로 설정한다. $\delta$는 극히 작은 실패 확률을 나타낸다.

가장 널리 쓰이는 기본 구현은 **DP-SGD**(Abadi et al., CCS 2016)이다. 핵심 두 단계는 그래디언트 클리핑과 가우시안 노이즈 주입이다.

```python
g_clipped = g / max(1, ||g||_2 / C)              # 민감도 제한
g_noisy   = g_clipped + N(0, sigma^2 * C^2 * I)  # 노이즈 주입
```

이때 누적 프라이버시 손실은 **moments accountant**(또는 Rényi DP)로 추적하여, 여러 step·라운드에 걸친 총 $(\varepsilon,\delta)$ 예산을 타이트하게 계산한다.

여기서 한 가지 구분이 중요하다. 원형 DP-SGD는 **개별 샘플(example-level)** 을 보호하지만, 연합학습에서는 "한 사용자의 데이터 전체"를 보호하는 **사용자/클라이언트 단위 DP(user-level DP)** 가 더 자연스럽다. Geyer et al.(2017)이 클라이언트 단위 DP를 처음 도입했고, McMahan et al.(2018)의 **DP-FedAvg**는 클라이언트 단위 업데이트 클리핑 + 가우시안 노이즈 + 클라이언트 샘플링을 통한 프라이버시 증폭(privacy amplification)으로 언어 모델에 적용된 대표 사례이다. 즉 연합학습 실무의 DP는 단순 DP-SGD가 아니라 이 user-level 변형을 가리키는 경우가 많다.

### 9-2. Homomorphic Encryption의 의미

HE의 핵심 성질은 **가법 동형성(additive homomorphism)** 이며, 다음 식으로 표현된다.

$$
\text{Enc}\!\left(\sum_{k=1}^{K} w_k\right) = \sum_{k=1}^{K} \text{Enc}(w_k)
$$

이 식의 의미는 **"암호화와 합산의 순서를 바꿔도 결과가 같다"** 는 것이다. 좌변은 "먼저 평문에서 합한 뒤 암호화"하는 경로이고, 우변은 "먼저 각자 암호화한 뒤 암호문 상태로 합산"하는 경로인데, 두 결과가 동일하다.

이 성질이 왜 중요한가? 일반 암호화에서는 합산을 하려면 복호화 키가 필요하므로, 서버에 키가 있어야 한다. 그러면 서버는 개별 $w_k$를 모두 볼 수 있게 되어 프라이버시가 깨진다. 반면 HE에서는 **서버는 복호화 키 없이도 암호문 상태에서 합산을 수행**할 수 있다.

> **즉, 서버는 각 병원의 가중치를 한 번도 직접 보지 않은 채로 합산할 수 있다.** 이 한 줄이 HE가 의료 다기관 연합학습의 표준 도구가 된 이유이다.

비유하자면, 각 병원이 자기 가중치를 같은 회사의 자물쇠로 잠근 상자에 넣어 서버에 보낸다. 서버는 자물쇠를 열 수 없지만, **잠긴 상자들을 합치는 것은 가능**하며, 합쳐진 상자를 열었을 때 안의 내용물이 자동으로 합산되어 있다. 서버는 합치는 작업만 하고, 합쳐진 결과는 키를 가진 측이 복호화하여 글로벌 모델 업데이트에 사용한다.

### 9-3. Gradient Compression의 동기와 방법

**Gradient Compression**은 통신 효율을 위한 기법으로, 압축 함수 $\mathcal{C}$를 적용한 그래디언트만 전송한다.

$$
\tilde{g}_k = \mathcal{C}\!\left(\nabla L_k(w_k)\right)
$$

왜 필요한가? 현대 신경망은 보통 수백만에서 수억 파라미터를 가진다. 100M 파라미터 모델을 32비트로 표현하면 한 라운드당 약 400MB이며, 수백 라운드면 디바이스당 수백 GB의 통신이 발생한다. **엣지 디바이스(스마트폰, IoT) 환경에서는 이것이 사실상 불가능**하므로 압축이 필수가 된다.

주요 압축 방법은 다음과 같다.

- **Top-k Sparsification**: 절댓값이 큰 상위 $k$개 성분만 전송. 100M 차원 중 상위 1%(=1M)만 보내면 통신량이 100배 감소한다.
- **Quantization**: 32비트를 8비트 또는 1비트(SignSGD; 부호만 전송)로 축소.
- **Error Feedback**: 버려진 그래디언트를 로컬에 누적해두었다가 다음 라운드에 합산하여 정보 손실을 보상한다.

Top-k와 Quantization을 결합하면 **수백 배 압축**이 가능하면서도 학습 정확도는 거의 그대로 유지된다 (Lin et al., ICLR 2018; Karimireddy et al., ICML 2019). 이는 프라이버시 자체보다는 통신 효율을 위한 기법이지만, 전송 정보를 줄임으로써 부수적인 보호 효과도 갖는다.

### 10-1. Google Gboard

수억 명 사용자의 입력 데이터를 휴대폰 밖으로 내보내지 않고, 학습된 모델 업데이트만 암호화하여 전송·집계한다. Federated Averaging 기반 모바일 FL의 **최초 구현 사례**로, Secure Aggregation과 DP를 결합해 사용자 노출을 막는다. 각 라운드는 다음 두 식으로 표현된다.

$$
\textbf{Secure Aggregation: } \quad \sum_k \tilde{w}_k = \sum_k w_k
$$

$$
\textbf{Differential Privacy: } \quad w_k^{DP} = w_k + \mathcal{N}(0, \sigma^2 I)
$$

### 10-2. Apple QuickType / Siri

로컬 업데이트 단계에서 FedAvg와 DP를 적용하고, 글로벌 집계 단계에서 Secure Aggregation으로 암호화된 상태의 업데이트를 모은다. **단계별 흐름을 명확히 정리**하면 다음과 같다.

**1단계 — Local update (각 클라이언트):**

$$
w_k^{t+1} = w_k^{t} - \eta \, \nabla L_k(w_k^{t})
$$

**2단계 — DP noise 추가 (서버 전송 직전):**

$$
\tilde{w}_k^{t+1} = w_k^{t+1} + \mathcal{N}(0, \sigma^2 I)
$$

**3단계 — Secure Aggregation (서버는 개별 $\tilde{w}_k$를 볼 수 없음):**

$$
w_{\text{global}}^{t+1} = \sum_{k=1}^{K} \frac{n_k}{N} \, \tilde{w}_k^{t+1}
$$

물결표(tilde, $\tilde{w}$) 기호로 DP가 적용된 가중치임을 구분한 점이 핵심이다. 서버는 SecAgg 프로토콜로 합산 결과만 수신하므로, 어떤 클라이언트의 $\tilde{w}_k$도 개별적으로 보지 못한다.

### 10-3. Intel + University of Pennsylvania (Healthcare)

병원 간 환자 영상 데이터를 직접 공유하지 않으면서, HE를 통해 **암호화된 채로 모델 업데이트를 합산**하여 글로벌 모델을 갱신한다. 단순 암호화는 복호화 후에만 연산이 가능하지만, HE는 암호화된 상태에서 덧셈·곱셈이 가능하다.

$$
\text{Enc}\!\left(\sum_{k=1}^{K} w_k\right) = \sum_{k=1}^{K} \text{Enc}(w_k)
$$

서버는 병원 A·B·C의 업데이트를 **복호화 없이** 합산하여 글로벌 모델을 만든다. 본 강의 청중의 연구 분야인 다기관 의료 영상 협업과 가장 직결되는 사례이다.

### 10-4. Amazon Alexa / IoT

엣지 디바이스에서 사용자 맞춤형 모델을 개선하되 민감 데이터를 저장하지 않는다. 디바이스 성능을 고려한 **Edge-aware scheduling**(충전 중·와이파이 연결·사용자 미사용 상태인 디바이스만 선정)과 **Gradient Compression**으로 제한된 통신 환경에 대응한다.

$$
w_{\text{global}}^{t+1} = w_{\text{global}}^{t} - \eta \sum_{k=1}^{K} \frac{n_k}{N} \cdot \mathcal{C}(g_k)
$$

여기서 $g_k = \nabla L_k(w^t)$, $\mathcal{C}$는 Top-k Sparsification과 Quantization을 결합한 압축 함수이다. 또한 한 디바이스의 데이터 분포가 **언어·문화별로 매우 다르다는 점**(예: "Play a song" / "Toca una canción" / "曲を再生する")이 자연스러운 **Non-IID 시나리오**를 만든다.

### 10-5. NVIDIA Clara / FLARE

다기관 의료 영상 분석에 특화된 플랫폼으로, BraTS 같은 뇌종양 분할 챌린지에서 여러 병원의 환자 데이터를 외부 유출 없이 통합 학습하는 데 사용되었다. **의료 FL의 사실상 표준 프레임워크**이며, 본 강의 청중의 실무 연구에 직접 적용 가능하다.

## 11. 실습 코드

### 11-1. 실습의 본질적 모순과 시뮬레이션의 의미

실습에 들어가기 전에 한 가지 모순을 정직하게 짚어야 한다. **연합학습의 본질은 데이터가 물리적으로 분산되어 있어야 한다는 것**이지만, 강의 실습에서는 한 사람(또는 한 서버)이 모든 데이터를 가지고 시뮬레이션할 수밖에 없다. "왜 우리가 3개 병원의 데이터를 모두 가진 채로 연합학습을 한다고 하는가?"라는 의문이 자연스럽게 든다.

이 모순은 숨기지 않고 정면으로 다룬다. **본 실습은 한 컴퓨터에 여러 클라이언트의 데이터가 모두 있는 상황이며, 이는 실제 연합학습의 정신과 정반대**이다. 우리는 **연합학습 알고리즘이 무엇을 하는지 이해하기 위한 시뮬레이션**을 하는 것이지, 실제 연합학습 시스템을 구축하는 것이 아니다. 핵심은 **"데이터를 옮기지 않는다고 가정하고, 모델 가중치만 주고받는 절차를 코드로 정확히 구현하는 것"** 이다. 이 점을 의식하면서 코드를 짜면, 시뮬레이션이 실제 시스템과 무엇이 같고 무엇이 다른지를 학습할 수 있다.

본 실습은 점진적인 3단계로 구성된다.

- **단계 1 — 단일 프로세스 시뮬레이션 (개념 이해)**: 한 파이썬 스크립트 안에서 K개의 클라이언트를 for문으로 시뮬레이션. FedAvg의 핵심인 "가중 평균" 절차와 Non-IID 효과를 손으로 구현한다.
- **단계 2 — Flower 다중 프로세스 (프레임워크 체험)**: 서버와 클라이언트를 별도 프로세스로 실행. 데이터는 각 클라이언트 프로세스의 메모리를 떠나지 않고 모델 가중치만 네트워크로 직렬화·전송되는, 실제 시스템과 동일한 구조를 체험한다.
- **단계 3 — 학생 노트북을 실제 클라이언트로 (시스템 체험, 선택)**: 강사 노트북에서 서버를, 학생 노트북에서 클라이언트를 실행. "내 데이터가 내 노트북을 한 번도 떠나지 않으면서 통합 모델이 만들어지는" 경험을 직접 체득한다.

대부분의 강의는 단계 1에서 멈추지만, 그 경우 학생들은 "이게 그냥 클라이언트별 mini-batch SGD와 뭐가 다른가?"라는 의문을 풀지 못한다. 가능하면 단계 2까지 한 번은 경험시키는 것이 좋다.

### 11-2. 단계 1 — 단일 프로세스 시뮬레이션 (MNIST)

FedAvg를 처음부터 구현하여 IID와 Non-IID 환경에서의 차이를 직접 확인한다. 모델은 단순한 CNN, 데이터는 MNIST를 사용한다.

#### 11-2-1. 데이터 분할

연합학습 실험의 핵심은 데이터를 어떻게 클라이언트별로 나누느냐이다. IID 분할은 단순 무작위로 균등하게 나누는 반면, Non-IID 분할은 각 클라이언트에 일부 클래스만 할당한다.

```python
import numpy as np
from torch.utils.data import Subset

def split_data_iid(dataset, K):
    """IID 분할: 무작위 균등 분배"""
    idx = np.random.permutation(len(dataset))
    return [Subset(dataset, idx[i::K]) for i in range(K)]

def split_data_non_iid(dataset, K, classes_per_client=2):
    """Non-IID 분할: 각 클라이언트가 일부 클래스만 보유"""
    labels = np.array([dataset[i][1] for i in range(len(dataset))])
    client_data = [[] for _ in range(K)]
    
    for c in range(10):
        idx_c = np.where(labels == c)[0]
        np.random.shuffle(idx_c)
        chosen = np.random.choice(K, classes_per_client, replace=False)
        splits = np.array_split(idx_c, classes_per_client)
        for client_id, split in zip(chosen, splits):
            client_data[client_id].extend(split.tolist())
    
    return [Subset(dataset, idx) for idx in client_data]
```

#### 11-2-2. 모델 정의

```python
import torch.nn as nn
import torch.nn.functional as F

class SimpleCNN(nn.Module):
    def __init__(self, n_classes=10):
        super().__init__()
        self.conv1 = nn.Conv2d(1, 16, 3, padding=1)
        self.conv2 = nn.Conv2d(16, 32, 3, padding=1)
        self.fc1   = nn.Linear(32 * 7 * 7, 64)
        self.fc2   = nn.Linear(64, n_classes)
    
    def forward(self, x):
        x = F.max_pool2d(F.relu(self.conv1(x)), 2)
        x = F.max_pool2d(F.relu(self.conv2(x)), 2)
        x = x.flatten(1)
        x = F.relu(self.fc1(x))
        return self.fc2(x)
```

#### 11-2-3. 로컬 학습 (FedAvg / FedProx 통합)

`mu`가 0이면 FedAvg, 양수이면 FedProx로 동작한다.

```python
import torch
import copy
from torch.utils.data import DataLoader

def local_train(model, global_state, dataloader, epochs=2, lr=0.01, mu=0.0):
    """로컬 SGD. mu>0이면 FedProx(proximal term)."""
    opt = torch.optim.SGD(model.parameters(), lr=lr)
    model.train()
    for _ in range(epochs):
        for x, y in dataloader:
            opt.zero_grad()
            loss = F.cross_entropy(model(x), y)
            if mu > 0:  # FedProx: 글로벌 모델 근방 규제
                prox = sum(((p - global_state[n]) ** 2).sum()
                           for n, p in model.named_parameters())
                loss = loss + (mu / 2) * prox
            loss.backward()
            opt.step()
    return model.state_dict()
```

#### 11-2-4. 글로벌 집계 (FedAvg / FedNova / FedOpt)

```python
def agg_fedavg(local_states, weights):
    """FedAvg의 핵심: 가중 평균  w_global = Σ p_k w_k"""
    total = sum(weights)
    avg = copy.deepcopy(local_states[0])
    for key in avg:
        avg[key] = sum(local_states[k][key] * (weights[k] / total)
                       for k in range(len(local_states)))
    return avg


def agg_fednova(global_state, local_states, weights, tau):
    """로컬 step 수(tau_k)로 정규화 후 평균."""
    total = sum(weights)
    p = [w / total for w in weights]
    tau_eff = sum(p[k] * tau[k] for k in range(len(tau)))
    new = copy.deepcopy(global_state)
    for key in new:
        norm_delta = sum(
            p[k] * (global_state[key] - local_states[k][key]) / tau[k]
            for k in range(len(local_states))
        )
        new[key] = global_state[key] - tau_eff * norm_delta
    return new


def agg_fedopt(global_state, local_states, weights, m, v,
               lr=0.01, b1=0.9, b2=0.99, eps=1e-3):
    """서버측 Adam (FedAdam). 유사 그래디언트 Δ_t를 적응적으로 적용."""
    total = sum(weights)
    p = [w / total for w in weights]
    new = copy.deepcopy(global_state)
    for key in new:
        delta = sum(p[k] * (local_states[k][key] - global_state[key])
                    for k in range(len(local_states)))
        m[key] = b1 * m[key] + (1 - b1) * delta
        v[key] = b2 * v[key] + (1 - b2) * delta ** 2
        new[key] = global_state[key] + lr * m[key] / (v[key].sqrt() + eps)
    return new
```

#### 11-2-5. 한 라운드 실행

```python
def federated_round(global_model, client_datasets, method="fedavg",
                    local_epochs=2, **kw):
    g_state = global_model.state_dict()
    local_states, weights, tau = [], [], []
    
    for dataset in client_datasets:
        m = copy.deepcopy(global_model)
        loader = DataLoader(dataset, batch_size=32, shuffle=True)
        mu = kw.get("mu", 0.0) if method == "fedprox" else 0.0
        local_states.append(local_train(m, g_state, loader,
                                         epochs=local_epochs, mu=mu))
        weights.append(len(dataset))
        tau.append(len(loader) * local_epochs)  # 로컬 step 수

    if method in ("fedavg", "fedprox"):
        new_state = agg_fedavg(local_states, weights)
    elif method == "fednova":
        new_state = agg_fednova(g_state, local_states, weights, tau)
    elif method == "fedopt":
        new_state = agg_fedopt(g_state, local_states, weights,
                                kw["m"], kw["v"])
    global_model.load_state_dict(new_state)
    return global_model
```

#### 11-2-6. 전체 학습 루프 (IID vs Non-IID 비교)

```python
from torchvision import datasets, transforms

def run_federated_learning(num_clients=10, num_rounds=20,
                            non_iid=False, method="fedavg"):
    transform = transforms.Compose([transforms.ToTensor()])
    train_data = datasets.MNIST('./data', train=True, download=True,
                                 transform=transform)
    test_data  = datasets.MNIST('./data', train=False, transform=transform)
    
    if non_iid:
        client_datasets = split_data_non_iid(train_data, num_clients)
        print("Non-IID 분할로 학습 시작")
    else:
        client_datasets = split_data_iid(train_data, num_clients)
        print("IID 분할로 학습 시작")
    
    global_model = SimpleCNN()
    test_loader = DataLoader(test_data, batch_size=256)
    
    for t in range(num_rounds):
        global_model = federated_round(global_model, client_datasets,
                                        method=method, local_epochs=2)
        global_model.eval()
        correct = 0
        with torch.no_grad():
            for x, y in test_loader:
                correct += (global_model(x).argmax(1) == y).sum().item()
        acc = correct / len(test_data)
        print(f"Round {t+1:2d} | Test Accuracy: {acc:.4f}")
    
    return global_model
```

이 코드를 IID와 Non-IID로 각각 실행해 보면, **IID 환경에서는 빠르게 95% 이상의 정확도에 수렴하지만, Non-IID 환경에서는 FedAvg의 학습이 불안정하고 최종 정확도도 낮아진다**. 반면 `method="fedprox"`(mu=0.01) 또는 `method="fednova"`로 바꾸면 Non-IID에서도 안정적인 수렴을 관찰할 수 있다. **"왜 FedProx·FedNova·FedOpt가 등장했는가"를 직접 손으로 확인하는 것이 단계 1의 가장 큰 교육 효과**이다.

### 11-3. 단계 1 보강 — DLG 데모로 "프라이버시 환상" 깨기

학생들이 흔히 빠지는 오해는 "연합학습 = 자동으로 프라이버시 보장"이다. 이 환상을 깨는 데 가장 효과적인 것은 **Deep Leakage from Gradients (DLG)** 데모이다. 그래디언트만 가지고 원본 이미지를 거의 완벽히 복원할 수 있음을 직접 보여주면, 학생들은 자연스럽게 "그러면 왜 DP·SecAgg·HE 같은 추가 기법이 필요한가"를 이해하게 된다.

핵심 아이디어는 단순하다. 진짜 그래디언트 $\nabla L(w; x^*, y^*)$를 알고 있을 때, 가상의 입력 $(x', y')$를 학습 가능 변수로 두고 그래디언트가 일치하도록 최적화하면 $(x', y')$가 원본 $(x^*, y^*)$로 수렴한다.

```python
# DLG의 핵심 골격 (개념 코드)
x_fake = torch.randn_like(x_real, requires_grad=True)
y_fake = torch.randn(num_classes, requires_grad=True)
optimizer = torch.optim.LBFGS([x_fake, y_fake])

target_grad = compute_gradient(model, x_real, y_real)  # 서버가 받은 그래디언트

for it in range(300):
    def closure():
        optimizer.zero_grad()
        fake_grad = compute_gradient(model, x_fake, y_fake.softmax(0))
        # 진짜 그래디언트와의 차이를 최소화
        loss = sum(((fg - tg) ** 2).sum() for fg, tg in zip(fake_grad, target_grad))
        loss.backward()
        return loss
    optimizer.step(closure)
# 300 step 후 x_fake가 x_real을 거의 복원
```

시각적 충격이 매우 크기 때문에, 강의 운영상 단계 1과 단계 2 사이의 30분 정도를 이 데모에 배분하는 것을 권장한다.

### 11-4. 단계 2 — Flower 다중 프로세스 (실제 시스템에 가까운 체험)

실무에서는 직접 구현보다 검증된 프레임워크를 사용한다. 가장 배우기 쉬운 것은 **Flower**(flower.dev)이다. 단계 1과 본질적으로 다른 점은 다음 세 가지이다.

- **서버와 클라이언트가 별도 프로세스**로 실행되므로, 모델 가중치는 실제로 직렬화되어 gRPC 등 네트워크 프로토콜로 전송된다.
- 데이터는 각 클라이언트 프로세스의 메모리만 접근한다. 비록 한 컴퓨터 안이지만 "**데이터는 클라이언트 프로세스를 떠나지 않는다**"는 분리가 코드 수준에서 강제된다.
- 클라이언트가 동시에 학습하고 서버가 비동기로 집계하는 등 실제 시스템의 패턴을 체험한다.

```python
import flwr as fl

class MnistClient(fl.client.NumPyClient):
    def __init__(self, model, train_data, test_data):
        self.model = model
        self.train_data = train_data
        self.test_data = test_data
    
    def get_parameters(self, config):
        return [val.cpu().numpy() for _, val in self.model.state_dict().items()]
    
    def fit(self, parameters, config):
        self.set_parameters(parameters)
        train(self.model, self.train_data, epochs=2)
        return self.get_parameters({}), len(self.train_data), {}
    
    def evaluate(self, parameters, config):
        self.set_parameters(parameters)
        loss, accuracy = test(self.model, self.test_data)
        return float(loss), len(self.test_data), {"accuracy": accuracy}

# 서버 실행
fl.server.start_server(
    server_address="0.0.0.0:8080",
    config=fl.server.ServerConfig(num_rounds=10),
    strategy=fl.server.strategy.FedAvg(),  # FedProx, FedAdam 등으로 교체 가능
)

# 클라이언트 실행 (별도 터미널 또는 별도 머신에서)
fl.client.start_numpy_client(server_address="server-ip:8080",
                              client=MnistClient(...))
```

### 11-5. 단계 3 — 학생 노트북을 실제 클라이언트로 (선택)

가장 인상적인 실습이다. 강사 노트북에서 Flower 서버를 실행하고, 같은 네트워크에 있는 학생 노트북들이 클라이언트로 접속한다. 각 학생은 자신의 노트북에 있는 데이터로 로컬 학습하며, 모델 가중치만 서버로 전송한다. 학생들은 자기 노트북의 데이터를 강사에게 한 번도 보내지 않는다.

예시 시나리오: 각 학생이 자신의 손글씨 이미지를 30장씩 미리 촬영 → 학생마다 자신의 손글씨로 로컬 학습 → 연합학습으로 전체 학생의 손글씨 패턴을 통합한 분류기 생성 → 마지막에 한 학생이 새로 쓴 글씨를 모든 모델이 인식해보기.

이 한 번의 경험이 "**내 데이터가 한 번도 내 노트북을 떠나지 않는데, 다른 학생들의 데이터까지 활용한 모델이 만들어지는**" 연합학습의 본질을 강렬하게 전달한다.

### 11-6. 흔히 빠지는 함정과 피하는 법

연합학습 실습을 운영할 때 주의할 점은 다음과 같다.

- **단계 1에서 멈추지 않기**: 가능하면 단계 2까지 한 번은 경험시킨다. 그렇지 않으면 "이게 그냥 클라이언트별 SGD와 뭐가 다른가?"라는 의문이 남는다.
- **DLG 데모 포함**: "연합학습 = 자동 프라이버시"의 환상을 깨지 않으면 DP·SecAgg·HE가 왜 필요한지 학생이 체감하지 못한다.
- **Non-IID 효과 반드시 시연**: IID에서만 잘 되는 것은 본질적 가치를 보여주지 못한다. 같은 코드로 두 시나리오를 비교 실행.
- **모델·데이터 크기는 작게 시작**: CIFAR-100 + ResNet50으로 시작하면 GPU·메모리 문제로 시간을 다 쓴다. MNIST + 간단한 CNN으로 알고리즘 본질을 완전히 이해한 후 확장.

## 12. 인공지능 신뢰성과의 연결

연합학습은 본 강의의 다른 신뢰성 모듈들과 본질적으로 얽혀 있다.

- **프라이버시** 측면에서는 연합학습 자체가 가장 직접적인 해법이며, DP와 Secure Aggregation의 결합으로 강화된다.
- **공정성** 측면에서는 클라이언트 간 데이터 분포가 다르기 때문에 글로벌 모델이 일부 그룹에 편향될 수 있어, group fairness 연구가 활발히 진행되고 있다.
- **강건성** 측면에서는 악의적 클라이언트가 잘못된 가중치를 전송하는 **Byzantine attack**과 모델 포이즈닝에 대한 방어가 핵심 연구 주제이다.
- **불확실성** 측면에서는 클라이언트 간 의견 불일치가 자연스러운 epistemic uncertainty의 측정 도구로 활용될 수 있다.

이러한 의미에서 연합학습은 **인공지능 신뢰성의 네 기둥을 모두 한 번에 다루는 종합 케이스**라 할 수 있으며, 본 강의의 마지막 모듈로 적합하다.

## 13. 요약

- **연합학습(Federated Learning)** 은 데이터 프라이버시를 **보장**하면서도 데이터를 **활용**하는 학습 패러다임이다. 중앙 서버에 데이터를 모으지 않고, 각 클라이언트가 학습한 모델만 받아 통합한다.
- 각 사용자/기관에서 학습된 모델은 **로컬 모델(local model)**, 이를 받아 통합한 모델은 **글로벌 모델(global model)** 이다.
- 데이터 보안 문제로 협력을 꺼리던 기관/기업/집단을 설득하기에 적합하며, 다양한 기관의 참여로 데이터가 증가하여 **모델 성능을 높이면서도 데이터 유출이 없다**는 점이 핵심 장점이다.
- 한 라운드는 **로컬 업데이트(client update)** 와 **글로벌 업데이트(server aggregation)** 로 구성되며, 대표 기법은 **FedAvg, FedProx, FedNova, FedOpt** 이다.
- 통신 구조(중앙집중/탈중앙화)와 데이터 분포(수평/수직/FTL)라는 두 축으로 프레임워크를 분류할 수 있다.
- 연합학습 자체로는 프라이버시가 완전 보장되지 않으며, **DP·Secure Aggregation·HE·Gradient Compression** 같은 기법과 결합하여 실무에 적용된다.
- 실제 빅테크 서비스(Google·Apple·Intel·Amazon·NVIDIA)는 FedAvg에 도메인 특성에 맞는 프라이버시·통신 효율 기법을 결합하여 운영한다.
- 실습은 **단일 프로세스 시뮬레이션 → Flower 다중 프로세스 → 학생 노트북 실제 클라이언트화**의 3단계로 진행하며, "데이터를 한 곳에 두고 시뮬레이션하는 모순"을 정직하게 명시한 뒤 진행한다.
