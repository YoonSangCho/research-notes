# 의료데이터 예측 모델링 프롬프트 템플릿

> 예측 모델링 작업 시 LLM 프롬프트로 즉시 활용 가능한 레퍼런스. 필요한 섹션을 복사하여 사용.

---

## 1. 데이터 전처리 및 불균형 대응

### 1.1 클래스 불균형 전략 (분류 문제)

| 구분 | 기법 |
|---|---|
| **오버샘플링** | SMOTE, Borderline-SMOTE, ADASYN, SVM-SMOTE, Random Oversampling |
| **언더샘플링** | Random Undersampling, Tomek Links, Edited Nearest Neighbors (ENN), NearMiss |
| **하이브리드** | SMOTE-Tomek, SMOTE-ENN |
| **알고리즘 수준** | `class_weight='balanced'`, `scale_pos_weight` 조정, Focal Loss |
| **임계값 조정** | Youden's J statistic, F1-optimal threshold |

> ⚠️ **데이터 누수 방지**: 오버샘플링은 학습 데이터에만 적용. CV pipeline 내부에서 fold별 적용 필수.

### 1.2 결측치 / 스케일링 / 인코딩

- **결측치**: Mean / Median / Mode imputation, KNN Imputer, MICE (Iterative Imputer), 결측 자체를 feature로 활용
- **스케일링**: StandardScaler, MinMaxScaler, RobustScaler *(이상치 많은 의료데이터 권장)*
- **인코딩**: One-Hot, Label, Target, Ordinal Encoding

## 2. 전통 ML 모델군 (Task별 선택)

| 카테고리 | 모델 |
|---|---|
| **선형** | Logistic / Linear Regression, Ridge (L2), Lasso (L1), ElasticNet (L1+L2), SGDClassifier / SGDRegressor |
| **비선형** | KNN, SVM/SVR (RBF, Polynomial), Naive Bayes (Gaussian, Bernoulli — 분류), Gaussian Process |
| **트리 기반** | Decision Tree, Random Forest, Extra Trees |
| **부스팅** | AdaBoost, Gradient Boosting, **XGBoost / LightGBM / CatBoost** *(tabular SOTA baseline)*, HistGradientBoosting |

## 3. 데이터 유형별 최신 딥러닝 / Foundation Models

> 2024–2026 최신 SOTA 중심. Foundation Model은 zero-shot / few-shot / fine-tuning 모두 가능.

|---|---|
| **MLP 계열** | MLP, ResNet-Tabular, GRANDE |

## 4. Foundation Model 활용 전략

| 전략 | 설명 | 사용 시점 |
|---|---|---|
| **Zero-shot** | 사전학습 모델 그대로 추론 | 데이터 매우 적음, 빠른 베이스라인 |
| **Few-shot / In-context** | 프롬프트에 예시 포함 (TabPFN, LLM) | 샘플 < 1K |
| **Linear Probing** | 특징 추출 후 linear classifier | 빠른 검증, low compute |
| **LoRA / QLoRA / DoRA** | 저랭크 adapter 학습 (PEFT) | 중간 규모, GPU 제한적 |
| **Full Fine-tuning** | 전체 가중치 학습 | 충분한 데이터 + 도메인 차이 큼 |
| **Prompt Tuning / Prefix Tuning** | 입력 prompt만 학습 | 대규모 LLM, 극저자원 |

> 🛠️ **PEFT 라이브러리**: HuggingFace `peft`, `unsloth` *(2x 속도)*, `axolotl`, `LLaMA-Factory`

## 5. 반복 실험 설계

### 5.1 Random Seed 반복 분할
- **Seed 리스트**: `[42, 123, 2024, 7, 17, 31, 99, 256, 512, 1024]` (최소 10회 권장)
- 각 seed마다 train/valid/test split → 학습 → 평가 → 결과 누적 → **평균 ± 표준편차** 보고

### 5.2 Cross-Validation 전략

| 데이터 특성 | CV 방식 |
|---|---|
| 분류 | StratifiedKFold (k=5 또는 10) — 클래스 비율 유지 |
| 회귀 | KFold (k=5 또는 10) |
| 시계열 | TimeSeriesSplit |
| 환자 단위 | GroupKFold (환자 ID 기준, 누수 방지) |
| 튜닝 + 일반화 동시 평가 | Nested CV |

### 5.3 하이퍼파라미터 튜닝
GridSearchCV / RandomizedSearchCV / **Optuna (Bayesian Optimization, 권장)** / Ray Tune / Weights & Biases Sweeps

## 6. 성능 평가 지표

### 6.1 Binary Classification
AUROC, **AUPRC** *(불균형 데이터 핵심 지표)*, Accuracy, Precision, Recall (=Sensitivity), Specificity, F1-score, NPV, PPV, Youden's J Index (= Sensitivity + Specificity − 1), Brier Score *(확률 캘리브레이션)*, Confusion Matrix

### 6.2 Multi-class Classification
AUROC (macro / weighted / ovr / ovo), AUPRC (macro / weighted), Macro-F1 / Weighted-F1, Cohen's Kappa, MCC (Matthews Correlation Coefficient), Top-k Accuracy

### 6.3 Multi-label Classification
Hamming Loss, Subset Accuracy (Exact Match Ratio), Label-wise AUROC / AUPRC (per-label → macro/micro/weighted 평균), Sample-averaged F1 / Macro-F1 / Micro-F1, Coverage Error, Ranking Loss, LRAP (Label Ranking Average Precision)

### 6.4 Regression
RMSE, MAE, MAPE, R², Adjusted R², Pearson / Spearman Correlation, **C-index (Concordance Index)** *(생존분석 / 순위 예측)*

### 6.5 Segmentation (의료영상)
**Dice Score (DSC)**, IoU / Jaccard, **HD95 (Hausdorff Distance 95%)**, ASSD (Average Symmetric Surface Distance), NSD (Normalized Surface Distance), Volumetric Similarity

### 6.6 Survival
**C-index (Harrell's / Uno's)**, Time-dependent AUC, Integrated Brier Score (IBS), Log-rank test

## 9. 코드 생성 요청 시 명시 항목

프롬프트 사용 시 아래 블록을 채워서 전달:

```
[데이터 정보]
- 데이터 유형: (tabular / time-series / 2D image / 3D medical (CT/MRI) / clinical text / multimodal)
- 파일 경로 / 디렉토리:
- Target 변수명:
- Task 종류: (binary / multi-class / multi-label / regression / segmentation / survival / forecasting)
- 샘플 수 / 피처 수 (또는 이미지 크기, 시퀀스 길이):
- 클래스 불균형 비율:
- 특수 컬럼: (환자 ID, 시간변수, 그룹변수 등)

[실험 조건]
- CV 방식: (StratifiedKFold k=5 / GroupKFold / TimeSeriesSplit / Hold-out)
- Random seed 반복 횟수: (예: 10회)
- 적용할 불균형 기법: (SMOTE / class_weight / Focal Loss / 둘 다 비교)
- 포함할 모델군:
    · Tabular: (부스팅 3종 / TabPFN v2 / FT-Transformer / 전체)
    · Time-series: (LSTM / PatchTST / iTransformer / Chronos zero-shot)
    · 2D Image: (ConvNeXt v2 / Swin v2 / DINOv2 fine-tune / MedSAM)
    · 3D Medical: (nnU-Net v2 / Swin UNETR / TotalSegmentator)
    · Clinical Text: (PubMedBERT / Meditron / GatorTron)
- Foundation Model 활용: (zero-shot / linear probe / LoRA / full fine-tune)
- 하이퍼파라미터 튜닝: (Optuna / GridSearch / 기본값)

[출력 형식]
- 결과 테이블 (CSV / Excel) — mean±std + 95% CI
- 시각화: (ROC / PR / Calibration / Grad-CAM / SHAP / Dice 분포)
- 해석: (SHAP / Grad-CAM / Attention map / DCA)
- 통계 검정: (paired t-test / DeLong / Wilcoxon)

[하드웨어/환경]
- GPU: (예: A100 80GB x1, RTX 4090, CPU only)
- 프레임워크 선호: (PyTorch / Lightning / MONAI / HuggingFace / sklearn)
- 학습 시간 제약: (예: 24시간 이내)
```

> 위 항목을 채워 전달하면 데이터 유형에 맞는 pipeline (sklearn / PyTorch Lightning / MONAI / HuggingFace) + CV + 평가 + 해석까지 한 번에 돌아가는 코드를 생성. 특정 섹션(3D segmentation 전용, Foundation Model fine-tuning 등) 구체화도 가능.
