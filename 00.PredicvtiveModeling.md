# 의료데이터 예측 모델링 프롬프트 템플릿

> 예측 모델링 작업 시 LLM 프롬프트로 즉시 활용 가능한 레퍼런스. 필요한 섹션을 복사하여 사용.

---

## 1. 데이터 전처리 및 불균형 대응

### 1.1 클래스 불균형 전략 (분류 문제)

| 구분 | 기법 |
|---|---|
| **오버샘플링** | **SMOTE** *(소수 클래스 k-NN 사이 보간 합성)*, Borderline-SMOTE, ADASYN *(어려운 샘플 가중 합성)*, SVM-SMOTE, Random Oversampling |
| **언더샘플링** | Random Undersampling, Tomek Links *(경계 다수 샘플 제거)*, Edited Nearest Neighbors (ENN), NearMiss |
| **하이브리드** | SMOTE-Tomek, SMOTE-ENN *(합성 후 노이즈 정리)* |
| **알고리즘 수준** | `class_weight='balanced'`, `scale_pos_weight` 조정, **Focal Loss** *(easy negative 다운웨이팅; Lin et al., ICCV 2017)* |
| **임계값 조정** | Youden's J statistic, F1-optimal threshold |

> ⚠️ **데이터 누수 방지**: 오버샘플링은 학습 데이터에만 적용. CV pipeline 내부에서 fold별 적용 필수.

### 1.2 결측치 / 스케일링 / 인코딩

- **결측치**: Mean / Median / Mode imputation, KNN Imputer, MICE (Iterative Imputer — chained equations로 다변량 추정), 결측 자체를 feature로 활용 *(MNAR일 때 missingness indicator)*
- **스케일링**: StandardScaler, MinMaxScaler, RobustScaler *(이상치 많은 의료데이터 권장)*
- **인코딩**: One-Hot, Label, Target, Ordinal Encoding

## 3. 데이터 유형별 최신 딥러닝 / Foundation Models

> 2024–2026(mid) 최신 SOTA 중심. Foundation Model은 zero-shot / few-shot / fine-tuning 모두 가능. 모델 버전·라이선스는 빠르게 바뀌므로 사용 전 공식 repo 재확인 권장.

### 3.1 Tabular Data

| 분류 | 모델 |
|---|---|
| **MLP 계열** | MLP, ResNet-Tabular *(Gorishniy et al., NeurIPS 2021)*, GRANDE |
| **Attention 기반** | TabNet *(sequential attention feature 선택)*, TabTransformer, **FT-Transformer** *(Feature Tokenizer+Transformer, 강한 DL 베이스라인)*, SAINT, AutoInt |
| **Tree-NN 하이브리드** | NODE, GrowNet, DeepGBM |
| **생성 기반** | TabDDPM (diffusion), CTGAN, TVAE *(데이터 증강용)* |
| **Foundation Model** | **TabPFN v2** *(Nature 2025; prior-fitted network, gradient 없이 단일 forward in-context 추론. ~10K행/500열 SOTA)*, **TabPFN-2.5 / TabICL** *(~100K행·2K열로 확장)*, TabuLa-8B (LLM-based), CARTE, XTab |
| **LLM-as-tabular** | Claude / GPT 직접 prompting (few-shot), Serialize-and-prompt 방식 |

> 🎯 **권장**: 샘플 수 < 10K → TabPFN v2 우선 시도(분류·회귀 모두). 더 큰 데이터 → TabPFN-2.5/TabICL 또는 FT-Transformer, SAINT + 부스팅 앙상블.
> 📌 TabPFN v2: Hollmann et al., *Accurate predictions on small data with a tabular foundation model*, Nature 637 (2025).

### 3.2 Time-Series (시계열 / EHR / 생체신호)

| 분류 | 모델 |
|---|---|
| **RNN 계열** | LSTM, GRU, Bidirectional LSTM, ConvLSTM |
| **CNN 계열** | 1D-CNN, TCN (Temporal Convolutional Network), InceptionTime, MiniRocket *(random convolution kernel, 매우 빠른 분류)* |
| **Transformer (long-horizon)** | Informer, Autoformer, FEDformer, **PatchTST** *(시계열을 patch 토큰화, channel-independent)*, **iTransformer** *(변수를 토큰으로 — attention을 채널 간에 적용)*, Crossformer, TimesNet |
| **State-Space Models (SSM)** | **Mamba** *(selective SSM, 선형 복잡도 긴 시퀀스)*, S4, S5, TimeMachine, Mamba-2 |
| **Decomposition** | N-BEATS, N-HiTS, **DLinear** *(단순 선형이 Transformer를 종종 능가 — 강력 베이스라인)*, NLinear |
| **Foundation Model** | **TimesFM 2.x** (Google, decoder-only), **Chronos / Chronos-Bolt → Chronos-2** (Amazon; 2의 다변량·공변량 지원, '25), **Moirai** (Salesforce), **Lag-Llama**, MOMENT, TimeGPT, Tiny Time Mixer (TTM) |

> 🎯 **권장**: 단변량 zero-shot forecasting → Chronos-2 / TimesFM 2.x. 다변량 supervised → iTransformer, PatchTST(+ DLinear 베이스라인 필수). EHR sequential → BEHRT 계열.

## 4. Foundation Model 활용 전략

| 전략 | 설명 | 사용 시점 |
|---|---|---|
| **Zero-shot** | 사전학습 모델 그대로 추론 | 데이터 매우 적음, 빠른 베이스라인 |
| **Few-shot / In-context** | 프롬프트에 예시 포함 (TabPFN, LLM) | 샘플 < 1K |
| **Linear Probing** | 특징 추출 후 linear classifier | 빠른 검증, low compute |
| **LoRA / QLoRA / DoRA** | 저랭크 adapter 학습 (QLoRA=4bit 양자화 결합, DoRA=크기·방향 분해) | 중간 규모, GPU 제한적 |
| **Full Fine-tuning** | 전체 가중치 학습 | 충분한 데이터 + 도메인 차이 큼 |
| **Prompt Tuning / Prefix Tuning** | 입력 prompt만 학습 | 대규모 LLM, 극저자원 |

> 🛠️ **PEFT 라이브러리**: HuggingFace `peft`, `unsloth` *(2x 속도)*, `axolotl`, `LLaMA-Factory`

## 5. 반복 실험 설계

### 5.1 Random Seed 반복 분할
- **Seed 리스트**: `[42, 123, 2024, 7, 17, 31, 99, 256, 512, 1024]` (최소 10회 권장)
- 각 seed마다 train/valid/test split → 학습 → 평가 → 결과 누적 → **평균 ± 표준편차** 보고

### 5.2 Cross-Validation 전략

| 데이터 특성 | CV 방식 |
|---|---|
| 분류 | StratifiedKFold (k=5 또는 10) — 클래스 비율 유지 |
| 회귀 | KFold (k=5 또는 10) |
| 시계열 | TimeSeriesSplit |
| 환자 단위 | GroupKFold (환자 ID 기준, 누수 방지) |
| 튜닝 + 일반화 동시 평가 | Nested CV |

### 5.3 하이퍼파라미터 튜닝
GridSearchCV / RandomizedSearchCV / **Optuna (Bayesian Optimization, 권장)** / Ray Tune / Weights & Biases Sweeps

## 6. 성능 평가 지표

### 6.1 Binary Classification
AUROC *(임계값 무관 순위 능력; 불균형 시 낙관적일 수 있음)*, **AUPRC** *(불균형 데이터 핵심 — 양성 희소 시 AUROC보다 민감)*, Accuracy, Precision, Recall (=Sensitivity), Specificity, F1-score, NPV, PPV, Youden's J Index (= Sensitivity + Specificity − 1), Brier Score *(확률 캘리브레이션 — 낮을수록 좋음)*, Confusion Matrix

### 6.2 Multi-class Classification
AUROC (macro / weighted / ovr / ovo), AUPRC (macro / weighted), Macro-F1 / Weighted-F1, Cohen's Kappa, MCC (Matthews Correlation Coefficient), Top-k Accuracy

### 6.3 Multi-label Classification
Hamming Loss, Subset Accuracy (Exact Match Ratio), Label-wise AUROC / AUPRC (per-label → macro/micro/weighted 평균), Sample-averaged F1 / Macro-F1 / Micro-F1, Coverage Error, Ranking Loss, LRAP (Label Ranking Average Precision)

### 6.4 Regression
RMSE, MAE, MAPE, R², Adjusted R², Pearson / Spearman Correlation, **C-index (Concordance Index)** *(생존분석 / 순위 예측)*

### 6.5 Segmentation (의료영상)
**Dice Score (DSC)**, IoU / Jaccard, **HD95 (Hausdorff Distance 95%)**, ASSD (Average Symmetric Surface Distance), NSD (Normalized Surface Distance), Volumetric Similarity

### 6.6 Survival
**C-index (Harrell's / Uno's)**, Time-dependent AUC, Integrated Brier Score (IBS), Log-rank test
