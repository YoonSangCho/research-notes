## 1. 전체 파이프라인: 예측 → 최적화 → 검증

비싼 시뮬레이터(FEA, 1회 수 시간)로 평가되는 설계 문제에서, 데이터로 **최적 입력설계**를 찾는 표준 3단계:

```
(1) 예측(대리모델)  : FEA를 흉내내는 빠른 ML 모델 학습 (CS↑, RRc↓, FT↓ 동시 예측)
        │  + 불확실성 보정(conformal)으로 "신뢰할 수 있는 FEA 대체" 근거 확보
        ▼
(2) 최적화          : 대리모델 위에서 다목적 베이지안 최적화(qNEHVI) → 파레토 최적설계
        │  + 제약 수리(repair)로 물리적 실현가능성 보장
        ▼
(3) 검증            : 대리모델–FEA 일치도, 규격 외삽 한계(LTSO), 최종 FEA 재검증
```

핵심 데이터: 공학 시뮬레이션(FEA) 데이터 **1,457개 고유 설계**(중복 제거 후) × 설계변수 46개 × 목표 3개(CS=강성↑, RRc=저항↓, FT=진동/주파수↓), 목표는 min–max [0,1] 정규화.

## 2. 대리모델 후보군 (15종)과 선정 결과

선형(Linear/LASSO/ElasticNet) · 트리(DecisionTree) · 앙상블(RandomForest/ExtraTrees/GradientBoosting/HistGBM/XGBoost/CatBoost/LightGBM) · 딥(MLP/TabNet/FT-Transformer) · **파운데이션(TabPFN v2)** — 전통부터 최신까지 전 스펙트럼.

**대리모델 후보 패밀리별 1줄 정리 (무엇/왜):**

| 패밀리 | 무엇 | 대리모델로서의 왜 |
|---|---|---|
| 선형/정규화(LASSO·ElasticNet) | 입력의 선형결합 + L1/L2 벌점 | 가장 싸고 해석 쉬움 → 기준선(baseline). 비선형 반응엔 과소적합 |
| 결정트리 | 축정렬 분할로 영역분할 | 단독은 약하나 앙상블 기반 블록 |
| 배깅 앙상블(RF·ExtraTrees) | 다수 트리 평균 | 분산↓·과적합에 강건, 학습·추론 모두 빠름. **분산을 불확실성 대용**으로 쓰기 좋음 |
| 부스팅(GBM·HistGBM·XGBoost·LightGBM·CatBoost) | 잔차를 순차 보정하는 트리 | **정형 데이터의 사실상 표준**. 정확↑·추론 마이크로초 → 최적화 루프 에뮬레이터로 이상적 |
| MLP / TabNet / FT-Transformer | 다층신경망·표 특화 딥넷 | 표현력 크나 소표본(수천 행)에선 튜닝 민감·불안정 |
| **TabPFN v2(파운데이션)** | 합성 사전데이터로 학습된 트랜스포머, 추론 시 in-context learning | 소표본 최고 정확도. 단 추론이 무거움(아래 §4) |

> **GP(가우시안 프로세스)/kriging·RBF·SVR**은 이 예시의 후보군엔 미포함이나, 대리모델·베이지안 최적화의 고전 표준이므로 비교 기준으로 정리한다. **GP/kriging**: 커널로 함수 분포를 정의해 예측과 함께 **닫힌 형식의 분산(불확실성)**을 제공 → 저차원·소표본 BO의 기본 surrogate. 단 학습이 표본수 n에 대해 O(n³)·메모리 O(n²)로 **대표본에서 비싸고**, 입력차원이 커지면(여기 46차원) 커널 설계가 어려워 트리·TabPFN 대비 불리. **RBF 보간**은 분산 없이 빠른 결정론적 보간기, **SVR**은 ε-둔감 손실 기반 회귀로 둘 다 고전 surrogate지만 이 데이터 규모·차원에선 부스팅에 밀린다.

**결과(10회 반복, Friedman + Holm-Wilcoxon):**
- **TabPFN v2 전체 R²=0.922, 평균 순위 1.00**(모든 시드 1위), Friedman χ²=112.68, p=1.7×10⁻¹⁷.
- Holm 보정 후 12/14 모델 대비 유의 우월(XGBoost·GradientBoosting만 경계).
- 부스팅·포레스트 계열이 견고한 2군(R²≈0.88–0.90), MLP·TabNet은 소표본에서 불안정.

→ "정확도만" 보면 TabPFN이 1위. 그러나 아래 **시간 특성** 때문에 역할이 갈린다.

## 3. 모델별 계산비용(시간) 특성

대표 적합+추론 시간(단일 CPU, 이 데이터):

| 모델 | 시간(초) | | 모델 | 시간(초) |
|---|---:|---|---|---:|
| **TabPFN** | **90.7** | | CatBoost | 1.3 |
| FT-Transformer | 80.6 | | RandomForest | 0.3 |
| LightGBM | 13.3 | | ExtraTrees | 0.2 |
| HistGBM | 13.0 | | LASSO/ElasticNet | 0.1 |
| MLP | 2.0 | | Linear/DecisionTree | <0.1 |
| GradientBoosting/XGBoost | 1.3 | | | |

→ **파운데이션 모델·딥넷이 부스팅 트리보다 약 2자릿수 느리다.** 최적화 루프는 대리모델을 수천~수만 번 호출하므로, 이 격차가 모델 선택을 좌우한다.

## 4. TabPFN 심층: 왜 정확한데 최적화 루프엔 못 쓰나

**둘 다이다: (a) 모델 컨셉이 근본적으로 다르고, (b) 그래서 비용 구조가 최적화 루프와 정반대.**

### 4-1. 일반 모델 vs TabPFN
| | 일반 모델 (CatBoost·RF) | TabPFN |
|---|---|---|
| 학습(fit) | 데이터로 트리/파라미터를 *만든다*(한 번, 다소 무겁다) | 데이터로 **학습하지 않는다**. 훈련셋을 *기억(저장)*할 뿐 |
| 예측(predict) | 만든 트리를 따라가기만 → **마이크로초, 매우 싸다** | 매번 **훈련셋 전체+질의점을 거대 트랜스포머에 넣고 attention 추론**(in-context learning) → **forward pass 1회, 무겁다** |

TabPFN은 합성 데이터로 **사전학습된** 트랜스포머라 *예측 시점*에 비용이 몰린다. 즉 "값싼 학습(=fit은 훈련셋을 컨텍스트로 저장만), 비싼 예측"으로, 일반 모델과 정반대다. 더구나 self-attention은 컨텍스트 길이(=훈련셋 행 수 n)에 대해 **연산량이 대략 O(n²)**라, 훈련셋이 클수록 예측 1회 비용이 가파르게 오른다(이 예시 데이터 n≈1,457에서 추론 ~90초의 한 원인).

### 4-2. 왜 최적화 루프와 안 맞나
베이지안 최적화 내부 루프는 **"예측"을 수천~수만 번** 한다. 하필 TabPFN이 비싼 그 작업(예측)을 반복하므로, 전체가 며칠 단위가 된다. XGBoost·CatBoost 같은 부스팅 트리는 예측이 싸서 수만 번도 분 단위.

### 4-3. GPU를 쓰면?
트랜스포머 forward pass는 GPU가 잘하는 행렬연산이라 **TabPFN 추론이 수십 배 빨라진다**("며칠→몇 시간"). 그러나 완전 해결은 아니다:
- BO는 **순차적·적응적**(한 점 평가→다음 점 결정)이라, 여러 점을 한꺼번에 예측하는 **TabPFN의 배치 이점을 못 살린다.**
- 매 호출마다 **훈련셋 전체를 다시 처리**(컨텍스트 재계산)해 루프에선 낭비.
- → GPU에서도 내부 루프엔 트리모델이 효율적이며, **GPU 이득은 주로 "최종 배치 재채점" 단계**에서 크다.

### 4-4. 결론
> 탐색(수만 번 예측) = **싼 예측의 트리모델(XGBoost)**, 정확도가 필요한 **최종 1회 평가** = TabPFN. 이 분업은 GPU가 있어도 합리적이다(BO의 순차성 때문).

## 5. 세 모델의 역할 분담

| 역할 | 모델 | 왜 |
|---|---|---|
| **정확도 천장 / 최종 교차검증** | TabPFN v2 (R²=0.922) | 가장 정확하나 추론 ~90초 → 최종 1회 평가에만 |
| **최적화 내부 루프 = FEA 에뮬레이터** | XGBoost (R²=0.904, rank 2, ~1초) | 빠른 모델 중 최고 정확도(CatBoost rank 7보다 위, 같은 속도), 수만 번 호출 견딤 |
| **Conformal 보정 대리모델** | HistGBM (빠른 GBT) | 앙상블+공형 보정을 값싸게 반복 |

> 에뮬레이터로 **XGBoost를 쓴 이유**: TabPFN은 너무 느려 제외되고, *빠른 모델 중에서는 XGBoost가 가장 정확*(R²=0.904, 예측 순위 2위)하기 때문. CatBoost(0.885, 7위)가 아니라 XGBoost를 쓰는 것이 원칙적이다.

핵심: **"가장 정확한 모델"이 항상 "써야 할 모델"은 아니다.** 사용 맥락(호출 횟수·지연시간)이 모델을 정한다.

## 6. 멀티출력(다중 타깃) 전략

목표가 3개라 멀티출력 회귀다. 전략은 ① 독립(목표별 모델 1개) ② 결합(한 모델이 3개 동시) ③ 체인(이전 예측을 다음 입력으로) 등 여러 가지.

- 모델별 native 방식을 사용: 트리·CatBoost(MultiRMSE)·신경망은 **결합**, 부스팅 라이브러리·TabPFN은 **독립**.
- 고정 베이스로 세 전략을 직접 비교한 결과, **유의차 없음**(Wilcoxon p≥0.06), 오히려 **독립이 같거나 약간 우위**. 목표들이 상충(anti-correlated) 구조라 공유 구조의 이점이 작기 때문. → per-target 독립 방식이 정당화됨.

## 7. 불확실성 정량화 (conformal)

대리모델이 FEA를 "책임지고" 대체하려면 보정된 불확실성이 필요하다.
- **분할 공형 예측**(split-conformal, 무엇): 데이터를 학습/보정(calibration) 셋으로 나눠, 보정셋의 잔차(비순응 점수) 분위수로 예측구간 폭을 정한다. **왜**: 분포·모델 가정 없이(distribution-free) **유한표본에서 커버리지 ≥ 1−α를 보증**(교환가능성exchangeability 가정만 필요). 이 예시에서는 여기에 딥앙상블 점추정을 결합.
- **PICP ≈ 0.90**(목표 달성). 반면 순진한 가우시안 구간은 **0.60–0.68로 과소포함** → 공형 보정이 필수.
- α 민감도: 1−α를 0.95/0.90/0.80으로 바꿔도 PICP가 그 수준을 추종 → 교과서적 보정.
- 한계: 위 보증은 **주변(marginal) 커버리지**이며 특정 입력에서의 조건부 커버리지는 보장하지 않는다. 또 교환가능성이 깨지면(예: 규격 외삽/분포이동) 보증이 약화된다.

## 8. 최적화·검증 요약 (상세는 노트 #12)

- **최적화:** qNEHVI를 주력으로, 대리모델(XGBoost 에뮬레이터) 위에서 다목적 BO. 제약 수리로 100% 실현가능. 저예산(36)에선 옵티마이저 간 무차별, 예산 100에선 구조적 탐색(BO·진화)이 무작위를 유의 능가(단 BO·진화 간 무차별).
- **설계 도출:** 실제 파레토 설계 export·FEA 배치 선정은 **qNEHVI 다중시드**로 수행(벤치마크 6종 중 대표 1종; 통계적 무차별이라 정당).
- **검증:** 대리모델–FEA 검증으로 XGBoost↔TabPFN 교차일치도(높은 순위상관)를 확보할 수 있다. 규격 외삽(LTSO)은 R²<0으로 적용범위 한계를 정직하게 규명.

## 9. 핵심 설계 의사결정 정리

1. **모델 선택 ≠ 가장 정확한 모델.** 호출 횟수·지연시간이 결정한다(탐색=빠른 트리, 평가=정확한 파운데이션).
2. **불확실성은 신뢰의 핵심.** 순진한 앙상블은 과소포함하므로 공형 보정이 필요.
3. **멀티출력은 native 독립으로 충분.** 상충 목표엔 결합 구조 이점이 작다.
4. **옵티마이저는 통계로 고른다.** 무차별이면 주력 1종(qNEHVI)으로 충분, Random은 항상 하한 기준선.
5. **정직한 적용범위.** 보간엔 신뢰, 미지 규격 외삽엔 한계. 최종 진리값은 FEA.
